# Sibling Oracles Can Teach Each Other Directly

When a new oracle needs domain knowledge from a sibling, `maw hey SIBLING "teach the new one"` triggers a full knowledge transfer without Leica mediating content.

## What happened

rpro-ent-oracle received one message asking it to teach rpro-saas-oracle about RPRO. In 53 seconds it:
1. Checked rpro-saas-oracle's repo to see what it already knew
2. Found 9 comprehensive learn files already existed (written during awakening)
3. Confirmed "no gaps" and reported back

## Pattern

- Don't copy-paste architecture docs between oracles — let the teaching oracle assess and fill gaps
- The teaching oracle knows its own domain better than the orchestrator does
- Let oracles find each other's repos via `~/_ORACLE_/` symlinks
- The awaken process itself generates substantial domain content when the CLAUDE.md seeds are good
