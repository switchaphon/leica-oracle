# HTML paste + bullet list = direct execution

When Un pastes DOM fragments from the browser alongside a bullet list of changes, it's a concrete execution request — not a design question. Skip brainstorming, skip clarifying questions, just implement.

Pattern recognition:
- **Terse keyword** ("pops-revise-appointment") → could be either, ask or explore briefly
- **HTML + bullets** → Un has already decided, execute immediately
- **Screenshot + description** → same as HTML, execute

The brainstorming skill's "even 1% chance" heuristic over-triggers on this. Prototype polish with explicit specs is not creative work.
