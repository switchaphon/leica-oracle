# Oracle Birth Ritual — The Complete Sequence

When birthing a new Project PM Oracle, follow this order:

1. **Read the project** — understand CLAUDE.md, tech stack, architecture
2. **Create repo** — `~/ghq/github.com/switchaphon/<name>-oracle/`
3. **Init git** — `git init`
4. **Create ψ/ brain** — inbox, outbox, plans, learn, memory/{resonance, learnings, retrospectives, collaborations, traces}
5. **Write ψ/.gitignore** — `active/`, `memory/logs/`, `learn/**/origin`
6. **Write resonance/oracle.md** — 5 principles, inherited from family
7. **Write CLAUDE.md** — identity, project context, service map, golden rules
8. **Register in fleet** — `~/.config/maw/fleet/<name>.json` with project_path and budded_from
9. **Update Leica's CLAUDE.md** — add to Project Oracle table
10. **Initial commit** — `awaken: <name>-oracle`
11. **maw wake** — start tmux session
12. **Send inbox letter** — introduce specialists, explain consult patterns
13. **Let it /learn** — deep-learn the project codebase
14. **Create GitHub remote** — push to switchaphon/<name>-oracle

The goodnight ritual: `tmux send-keys -t <session> "/rrr" Enter` — tell kids to write their diary.
