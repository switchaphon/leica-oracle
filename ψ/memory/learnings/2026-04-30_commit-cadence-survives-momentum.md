# Commit cadence must survive momentum

**Date**: 2026-04-30
**Source**: Day-wrap retrospective of April 29

The "commit early, shorter sessions" lesson was written at 12:15 on April 29 and violated by 13:00 the same day. The afternoon ran 5 hours across 4 sessions with zero commits — 1,800 lines of prototype code sitting in the working tree overnight.

The failure mode isn't forgetting the rule. It's that momentum makes stopping feel like friction. When 3 parallel Claude panes are moving fast, committing feels like interrupting flow. But that's exactly when it matters most — the more panes, the more fragile the state.

**Fix**: Make commits mechanical, not discretionary. After every successful browser verification, commit. Don't decide whether to commit — just do it. The decision was already made when you wrote the lesson the first time.
