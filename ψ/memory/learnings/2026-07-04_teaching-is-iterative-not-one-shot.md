# Teaching is iterative, not one-shot

**Date**: 2026-07-04
**Source**: rrr: leica-oracle
**Tags**: teaching, fleet, communication, pattern

## Lesson

Teaching an oracle a new concept (like codex team usage) is never one message. Plan for 2-3 rounds minimum:

1. **Round 1**: Core concept (what is it, decision tree, basic commands)
2. **Round 2**: Corrections from misconceptions exposed by round 1 (wrong model, wrong engine, wrong understanding)
3. **Round 3**: Hands-on guidance when they try and get stuck
