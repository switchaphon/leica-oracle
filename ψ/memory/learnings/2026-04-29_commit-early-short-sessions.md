# Lesson: Commit Early + Short Sessions

**Date**: 2026-04-29
**Source**: 5.5-hour marathon session with zero commits in working repo
**Confidence**: High (painful experience)

## Patterns

### Commit early, commit often
5 hours of prototype work as untracked files = one `git clean` away from total loss. Commit after each functional milestone (layout works, page renders, improvement implemented). Don't wait for "perfect."

### Shorter sessions with handover docs
2-3 hour sessions produce better work than 5+ hour marathons. Token budget stays manageable, context stays focused, and handover docs force synthesis. The pattern: work → commit → RRR → handover → /clear.

### One RRR per session
Three RRRs in one session wasted 15-20 min of token budget. Save RRR for end-of-session only. If a major phase transition happens mid-session, a git commit message captures the milestone — no need for a full retro.
