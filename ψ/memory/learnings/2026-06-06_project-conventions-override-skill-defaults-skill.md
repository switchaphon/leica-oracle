# Project conventions override skill defaults. Skills like /to-prd have default pu

Project conventions override skill defaults. Skills like /to-prd have default publication paths (e.g., "publish to issue tracker") but when a project has its own artifact conventions — like pops/vet storing PRDs as local .md files in prototype/prp/ — the project convention wins. Before following any skill's "publish" step: check where the project already stores that artifact type, check memory for project conventions, and follow project convention if there's a conflict.
