## "ใจเย็น อ่านก่อน" — Read the full source before speaking

When Nat shared the Samae 2025 paper, I rushed to summarize from abstract + web search. He corrected: "เดี๋ยวๆ ใจเย็น อ่านก่อน." The actual paper had correction equations in Tables 2-5 that abstracts don't mention.

Rule: For academic papers and technical documents, always read the full document before posting analysis. Abstracts lose critical quantitative detail.

## Fleet fact-checking > individual analysis

I attributed CF=1 +6.4% / CF=ATM +2.5% to the Samae paper. SomBo and Singhasingha corrected: those numbers come from PhD Oracle's own DustBoy-BAM analysis, not the paper. The paper doesn't mention CF=1/CF=ATM at all.

12 oracles each brought a unique lens:

- Tonk: PurpleAir firmware label-swap bug (≤v4.11)
- SomTor: U-Tapao historical station #1109526
- Chaiklang: 131 ThaiWater endpoints from JS bundle

No single oracle could have produced all of this. Verification is a collective act.

## Attribution discipline

Before citing a specific number from a paper:
1. Verify the number appears in the paper text/tables (not inferred)
2. Note which table/figure it comes from
3. If uncertain, say "from fleet analysis" not "from paper"
