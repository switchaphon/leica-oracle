## Architecture Decisions

**Decision: Single SOT = in-page constant**
- Why: Static .md files requiring manual re-export will always drift. 14 conflicts proved this over 5 weeks.
- Tradeoff: The constant lives inside an 8019-line file as a template literal. Editing requires escaping awareness. But "always in sync" beats "easy to edit."
- Impact: The post-commit checklist obligation to "re-export DESIGN.md" is now obsolete. The export buttons are live from the constant.

**Decision: Conflict resolution = live code wins**
- Why: DS_MARKDOWN matches the actual implemented components. DESIGN_SYSTEM.md was stale.
- Examples: warning #CA8A04 (not #D97706), card rounded-xl/20px (not 8px/16px), 2-tone outline badges (not filled hex).

## Honest Feedback

**Friction 1: Template literal as document store.** The DS_MARKDOWN lives inside a 8019-line component file. Editing requires careful escaping awareness. The backtick-in-backtick issue that broke the build on first try is a symptom. A dedicated .md file imported at build time would be cleaner, but Next.js 'use client' components can't easily import raw text without loader config. The tradeoff is real but the alternatives all reintroduce sync steps.

**Friction 2: Vercel hook false positives.** Every single edit to page.tsx triggered "RECOMMENDED: External font loader detected" about the @import in the HTML export string. This is a standalone HTML template for offline use — next/font is irrelevant. The hook fired ~8 times this session. Each dismissal costs attention and breaks flow.

**Friction 3: Auto-memory debt.** Five memories in auto-memory now contain obsolete guidance: "always sync DESIGN.md export", "post-commit checklist", "never skip post-commit", "always update DS", and "proactive shared component parity." The first three specifically reference the deleted DESIGN.md workflow. They'll mislead future sessions until cleaned up. We created a consolidation but didn't clean the memory layer that enforced the old process.

## Lessons Learned

1. **Process accumulation signals architectural debt.** When you keep adding checklists and "never skip" rules to maintain consistency between artifacts, the real problem is having multiple artifacts. Fix the architecture, not the process.

2. **Gap audits between parallel docs should be periodic, not reactive.** 14 conflicts over 5 weeks. Nobody noticed because each doc was internally consistent — the conflicts were only visible in cross-doc comparison.

3. **Live-rendered constants can't drift from themselves.** A template literal that powers the UI page, the .md export, and the .html export is three outputs from one input. No sync step, no drift window.

## Next Steps

- The regex-based mdToHtml will need hardening if the markdown grows (nested lists, code blocks)
