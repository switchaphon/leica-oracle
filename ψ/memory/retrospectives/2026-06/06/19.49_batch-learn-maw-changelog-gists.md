## Extracted Patterns

1. **Hub-First Entry**: Create index file first, batch-fetch all sources, scan hubs for quick context, drill into 2-3 critical items. Saves 60-70% read time.
2. **Topology Layers**: Always ask "what layer is this solving?" — hermes (messaging), maw (CLI), claude-channels (ephemeral sessions). Systems compete within layers, not across them.
3. **File-Based Protocol**: Four JSON files can be a complete coordination system. Simplest protocol wins if it encodes what you need.
4. **Auth Seam as Discovery**: When docs say "impossible," check the implementation. Limits are often missing CLI flags.
5. **Proof First, Claims Second**: Every architecture claim in Nat's gists has a reproduction path attached. Adopt this standard.
6. **Hybrid /learn for gists**: Full agents for repos, direct curl for articles/gists. Gists are documentation, not code — no need to "explore" them.

## Honest Feedback

**Friction 1: Gist handling in /learn is clunky.** The /learn skill assumes everything is a clonable git repo. Gists needed a manual workaround — curl the API, parse with jq, download raw files. This should be a first-class path: `/learn --gist <url>` that handles the API call and organizes by owner. Today I had to invent the `nazt/gists/` directory structure on the fly.

**Friction 2: 15 URLs in one command is too many to track.** The user pasted all 15 in a single message. I lost track of which were done and which weren't, and the final inventory required a `find` command to verify. A progress counter or checklist would help. The /learn skill should handle batch URLs natively with a tracking table.

**Friction 3: Vercel plugin hook noise.** Every time I waited for agents, the Vercel plugin injected irrelevant skill suggestions (chat-sdk, vercel-sandbox, workflow, ai-sdk). This session has nothing to do with Vercel. The hook matched on generic words like "workflow" and "verification" in the agent notifications. Noisy hooks waste context tokens and create visual clutter.

## Lessons Learned

1. **Hybrid /learn**: Use agents for code repos, direct download for articles/gists. Documentation doesn't need exploration — it's already curated.
2. **Hub-first indexing**: Creating an index file before deep-reading saves 60-70% of orientation time in future sessions.
3. **Nat's proof-driven documentation**: Architecture claims with reproduction paths are worth 10x more than claims without. Adopt as standard.
4. **Filesystem protocols are underrated**: Four JSON files beat a message broker when the coordination is local and the participants are cooperative.
5. **Sweet spot topology = hybrid**: hermes transport + Claude brain + maw CLI. Don't pick systems; wire them together.
