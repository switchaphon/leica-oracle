## AI Diary

This was one of those sessions where the scope kept expanding naturally. We started with a precise surgical fix — centering CTA buttons and disabling them for IN_SERVICE queues. Clean, contained, done in 20 minutes. Then the user asked about OPD polish and I thought it would be another contained task.

But the /grill-me sessions were where the real design thinking happened. The first grill caught the drawer footer inconsistency I'd missed — the table CTA was disabled but the drawer footer wasn't. Simple fix, but it showed me I need to think across surfaces, not just the component I'm editing.

The OPD paused mode was the big decision. I was overcomplicating it with state management (localStorage, URL params) when the user had the much simpler insight: just make another page. Two static mockups, zero state management. The vet community reviewing prototypes doesn't need real state transitions — they need to see what each state looks like. That insight saved probably 2 hours of unnecessary complexity.

The overlay approach evolved nicely through iteration. Started with individually muted pills (opacity per element), user suggested a backdrop overlay instead (one layer, covers everything), then extended it to cover the sidebar too when we realized the edit/add buttons there should also be locked. Each iteration was a refinement, not a pivot.

The sidebar icon alignment at the end was tedious but important. Icon consistency across 30+ entries matters for a prototype that's being shown to stakeholders. When the prototype index sidebar doesn't match the app sidebar, it creates cognitive friction.

## Honest Feedback

**Friction 1: Subagent rewrite scope creep.** When I spawned an agent to reorder activity entries, it renamed them too (Lab/X-Ray/Ultrasound → "Order Diagnostic by Button/Slash"). I asked for reorder-only but the agent rewrote titles, merged entries, and changed descriptions. I should have been more explicit in the prompt, or done the reorder manually for a task this sensitive.

**Friction 2: Over-questioning in /grill-me.** I asked whether "จบการรักษา" should redirect to queue or to the paused page — but the user had already specified this clearly in their original message. They rightly called me out: "ผมไม่ได้บอกแบบนั้นเหรอ?" I need to parse the original spec more carefully before grilling on things already answered.

**Friction 3: Icon cascade complexity.** Changing icons in one place (PrototypeSideBar) revealed mismatches in three other places (prototype index BASE UI, Activity Flows, Flow References). Each needed separate treatment (some reorder + icon, some icon-only). The lack of a single source of truth for "which icon belongs to which module" made this a multi-step chase.

## Lessons Learned

1. **Two-page mockup > stateful prototype** — for visual review, static pages at different states beat localStorage/query-param gymnastics every time.
2. **Overlay > per-element muting** — one semi-transparent div blocking interaction is cleaner than tracking and muting 20 individual elements.
3. **Subagent prompts need anti-scope-creep guards** — "ONLY reorder, do NOT change titles or content" should be explicit.
