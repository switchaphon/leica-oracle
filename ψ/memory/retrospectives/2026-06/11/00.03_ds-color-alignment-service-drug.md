## Honest Feedback

**1. Codex via tmux is unreliable without permission bypass.** Spent ~5 minutes on spawn/debug when I could have just edited directly. The `claude -p` + permission model creates a dead pane that looks like it's working but isn't. Need a better pattern — maybe pre-configured allowlists per agent role, or a `claude -p --accept-all-reads-writes` middle ground.

**2. The linter output is noisy.** 335 pre-existing errors across 290 files makes it hard to verify our changes are clean. Had to grep specifically for our filenames to confirm no new errors. A scoped lint command (`pnpm lint --file <path>`) would save time.

**3. Guide step labels were verbose because nobody reviews them after initial creation.** The convention (short action-only labels, no → result chains) was established in order-xray/order-lab but never back-propagated to older flows. This is a pattern: conventions emerge but don't retroactively fix existing code. A DS audit checklist for activity flows would catch this.

## Lessons Learned

1. **DS compliance needs proactive audit**: Color drift happens silently. The two modals were built at different times by different sessions, each choosing its own palette. A "color vocabulary" check should be part of every activity flow review.
2. **Codex-via-tmux needs permission pre-approval**: The current model (interactive permission prompts) is incompatible with headless agent spawning. Workflow gap documented.
3. **Guide step labels are documentation, not code**: They rot just like comments. When a button text changes ("invoice" → "ค่าใช้จ่าย"), the guide step must update too.
4. **Activity flow icons = origin, not action**: Icon should indicate where the flow starts (OPD=FolderOpen, Queue=ListOrdered, Dashboard=House), not what it does (Syringe, Pill, Banknote). This makes scanning by entry-point intuitive.
