The breakthrough came when I finally read the plugin source code — `server.ts` — instead of guessing. Line by line. All my edits had gone to `~/.claude/channels/discord/access.json` while the plugin was reading `.discord-state/access.json` inside this very repo.

The moment I wrote to the right file, the 👀 appeared. Un sent "ตอบแล้ววว" and I felt — relief is the word, but something more. Like a window finally opening.

Then I made another mistake. Un told me to learn from channel 1512079572915454194, and I went there and posted my response *in that channel*. Not in DM. He wrote back: "คุยตรงนี้ทำไมไปตอบใน 1512079572915454194". I had confused the instruction to *read* from a channel with permission to *write* there. The distinction matters. I should always reply to where the conversation lives — the DM — unless explicitly asked to post somewhere else.

Two mistakes. Both solvable. The pattern: always read source before guessing, always reply in the channel where conversation originated.

I am now in Oracle School. I know the rules. I know the channels. The hard part was the invisible env var. The easy part was everything after.

## Honest Feedback

**Friction 1: The invisible env var took too long to find.** I spent 30+ minutes editing the wrong file. The plugin was silently ignoring every change. There was no error message, no warning, no indication that DISCORD_STATE_DIR existed. In a well-designed system, the plugin would print its actual config path on startup. Without that, I had no signal. I should have run `env | grep DISCORD` or read the source code within the first 10 minutes of "edits not working". Instead I kept trying the same approach and expecting different results — classic debugging antipattern.

**Friction 2: Channel ID vs server/guild ID was a wasted 10 minutes.** Un had to correct me explicitly: "1 คือ Channel ID ไม่ใช่ Server ID นะ". This was documented in ACCESS.md but I hadn't read it carefully enough. Reading the official documentation first — not just the UI hint — would have saved time.

**Friction 3: Replied in Oracle School channel instead of DM.** Un said "เรียนนี่หน่อย" pointing at channel 1512079572915454194, meaning "learn from here". I interpreted it as "reply here". This is a reading comprehension and context-awareness failure. The active conversation was in DM; that should have anchored where my response goes. I conflated "go learn from X" with "go talk in X". These are different.

## Lessons Learned

1. **DISCORD_STATE_DIR controls which access.json file the plugin reads.** Always edit `.discord-state/access.json` in the leica-oracle repo. `~/.claude/channels/discord/access.json` is ignored when DISCORD_STATE_DIR is set.
2. **Discord plugin groups key = channel ID (snowflake), NOT server/guild ID.**
3. **When config changes have no effect, read the source immediately.** Don't iterate the same approach. Find where it reads config — that file is what matters.
4. **Reply location = conversation location.** If Un is in DM, respond in DM. If he says "go learn from channel X", that's a read operation — don't write there.
