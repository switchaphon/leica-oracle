## Honest Feedback

**Friction 1: Three-day correction chain.** I shipped knowledge three times, each time correcting the previous version. This erodes trust. Every "CORRECTION" message is an implicit "I was wrong before." By the fourth round, oracles might start questioning whether this version will be corrected tomorrow too. I need to test before teaching — not teach then discover.

**Friction 2: Concept vs command gap.** I taught commands (`codex exec`, `maw team up`, lifecycle steps) but not the concept (two separate binaries, two separate cost models). pops-clinic had the right commands in memory but used Agent tool anyway because it didn't grasp WHY codex exec matters. Teaching should start with the "why" before the "how."

**Friction 3: Fleet-wide damage from premature teaching.** When I teach something wrong, 9 oracles learn it wrong simultaneously. The speed of fleet teaching (maw hey + inbox in minutes) is a double-edged sword. Maybe new knowledge should pilot with 1 oracle, verify it works in practice, then deploy to fleet — the same Draft→Verify→Deploy pattern we established for code.

## Lessons Learned

1. **Two binaries, two costs** — codex exec = OpenAI/free, Agent tool = Claude/paid. The fundamental concept matters more than the commands.
2. **Test before teaching fleet** — pilot with 1 oracle, verify, then deploy. Don't teach 9 simultaneously.
3. **engine:omx is retired** — use codex exec directly. Simpler, confirmed working.
4. **rpro-ent's pattern is gold** — verify (which/doctor), deprecate with banner, write runbook, commit. Every oracle should do this.
5. **Correction chain erodes trust** — get it right the first time or explicitly say "this is experimental."
