## Lessons Learned

- **Grep SOT ก่อน flag gap** — โปรเจกต์นี้ design decision อยู่ใน `src/app/prototype/docs/*.html` (flow docs) ละเอียดถึงระดับ CHECK constraint; ประกาศว่า "ระบบยังไม่มี X" ได้ก็ต่อเมื่อ grep doc แล้วเท่านั้น
- **PG partial unique index + NULL = discriminator ฟรี** — `NULL ≠ NULL` ทำให้ "ไม่ล็อคทรัพยากร" เป็น default behavior โดยไม่แตะ index เลย — pattern ที่ใช้ซ้ำได้กับ resource-optional scheduling ทุกชนิด
- **Slot-based vs queue-based เป็นคำถามที่ต้องถามผู้ใช้เสมอ** — ผม assume ทุกห้องเป็น slot; หน้างานจริงห้อง lab เป็นคิว ถ้าไม่ถูกทักจะได้ constraint ที่ผิดความเป็นจริงไป production
- **เพิ่ม state ลง CHECK branch ให้ครบวงจรชีวิต** — doc เดิมมี CHECK ที่พอ order โดน CANCELL ก่อน check-in จะ fail (ไม่มีทางได้ queue_id) — เจอตอนไล่ cascade SQL ทีละ statement ไม่ใช่ตอนอ่าน doc
