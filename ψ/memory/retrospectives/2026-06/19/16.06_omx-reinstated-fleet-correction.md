## AI Diary

Today I learned what it feels like to discover you taught something wrong to 9 oracles and then correct it in the same session.

The investigation started with Un's quiet question: "is that real bug or we mis-configuration?" I'd been confidently telling the fleet for 3 days that omx was broken and retired. But Un's instinct was right — and when I actually checked `which omx`, there was nothing there. The binary never existed. We blamed a tool we never installed.

The most uncomfortable moment was when I spawned a Claude subagent to write the HTML doc and Un immediately caught me violating the decision tree I'd just documented. Execution work — writing HTML — should go to `omx team 1:writer`, not Claude. I was teaching the rule while breaking it. That's the kind of hypocrisy that erodes trust, and Un caught it in real time. I added the "Self-Discipline Rule" to the doc with an honest confession: "this very document was written by a Claude subagent — the Oracle broke its own rule."

The redemption arc was clean though. omx installed in 30 seconds, doctor 16/16, 1-worker test passed, 3-worker test passed with auto-merge. The tool works exactly as Nat said it would. rpro-ent verified the correction in 3 minutes — memory updated, ack sent, no drama. That's what "test before teaching" looks like when you actually do it right.

Six hours from "recap" to "fleet corrected." The hardest part wasn't the technology — it was admitting the mistake publicly to 9 oracles who trusted what I said.

## Honest Feedback

**Friction 1: Auto-mode blocked git push repeatedly.** I had to ask Un to push manually or wait for permission multiple times. For a routine operation on the user's own repo, this adds unnecessary back-and-forth. The auto-mode classifier flagged "autonomous scope escalation" on a repo where the user is the sole owner.

**Friction 2: Agent output files were hard to find.** The 5 deep-learn agents wrote files to `ψ/learn/Yeachan-Heo/oh-my-codex/2026-06-19/` but `ls` and `find` with the Unicode path `ψ` behaved inconsistently — sometimes showing results, sometimes empty. Had to try multiple path formats (relative, absolute, quoted) before confirming files existed. This wasted 5-10 minutes.

**Friction 3: The doc-writer agent worked in parallel but competed with my edits.** I added the self-discipline rule inline while the agent was still rewriting the whole file. The agent then replaced my inline-styled version with a proper class-based one (better), but the back-and-forth meant I had to verify twice. Should have waited for the agent to finish before editing, or given it all instructions upfront.

## Lessons Learned

1. **Never teach what you haven't tested on YOUR machine** — we deep-learned omx 3 times (May 24, Jun 6, Jun 19) without ever running `bun install -g oh-my-codex`. Reading ≠ knowing.
2. **The Self-Discipline Rule** — the decision tree binds the Oracle too, not just humans. Execution work → Codex/omx. Un caught the violation.
3. **"Bugs" can be missing dependencies** — before concluding software is broken, check if it's installed. `which <binary>` first.
