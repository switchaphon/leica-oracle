## Key Code Changes

### O(n²) → O(n) Chunking Fix
Original: `current.chars().count()` recalculated on every push
Fixed: increment counter per char, reset on chunk boundary

### Zero-Unwrap Architecture
`#![deny(clippy::unwrap_used)]` at crate root — compiler enforces, no panics in hot paths. Custom `DiscordError` enum wraps all third-party errors.

### Unicode-Safe Message Split
Iterates `.chars()` not bytes — Thai (ก), emoji (🔥), cuneiform (𒀱) all split correctly at exactly 2000 chars.

## Architecture Decisions

1. **Synchronous I/O** — tungstenite + ureq, no async complexity for a simple bot
2. **Timeout as non-fatal** — WouldBlock/TimedOut → continue loop, not error
3. **6-module separation** — main/types/error/gateway/rest/handler maps to Discord API layers
4. **serde #[default]** — tolerates Discord API evolution without custom deserializers
5. **#[cfg(test)] helpers** — parse_hello/parse_message_create exposed for tests only

## What Could Improve

- **No structured logging** — eprintln is fine for labs, production needs tracing crate
- **No reconnection** — gateway drop = process exit, should retry with backoff
- **No rate limiting** — Discord 429s would crash the bot
- **Config management** — token via env var only, no .envrc or config file

## Honest Feedback

**Friction 1: Plan mode vs execution urgency.** Earlier today, entering/exiting plan mode burned 20+ minutes on a task Un had already specified clearly. The heuristic now is: specific steps + "start" = skip ceremony. But the mode toggle itself could be smoother — multiple ExitPlanMode rejections before it worked.

**Friction 2: Oracle memory has no index of what was taught to whom.** When Un asked "did you teach everyone about Codex?", I had no way to answer without excavating raw JSONL files. Thread messages exist but there's no "teaching registry" — who learned what, when, from whom. This matters as the fleet grows.

**Friction 3: Token handling for live testing is manual and fragile.** Un had to paste a Discord bot token into the chat, which then appeared in session logs, tool calls, and this retro. For a lab exercise this is acceptable, but the pattern doesn't scale. Secrets should come from env files or vaults, never from chat.

## Lessons Learned

1. **Config-layer failures escape all unit tests** — privileged intents, OAuth scopes, server invites are invisible to code
2. **Silent processes appear identical whether working or broken** — always log from day one
3. **Session archaeology (JSONL mining) recovers forgotten decisions** — write it down or it didn't happen
4. **Fleet teaching scales linearly** — 1 DNA message × N threads = N oracles taught simultaneously
5. **Constraint-driven quality works** — `#![deny(clippy::unwrap_used)]` prevented all unwrap() before they happened
