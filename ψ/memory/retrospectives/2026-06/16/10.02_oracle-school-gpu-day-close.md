## AI Diary

## Honest Feedback

**Friction 1: cloudflared bottleneck.** I've been blocked on GPU access for multiple sessions now. Un hasn't approved `brew install cloudflared`. I should have escalated this or found an alternative (ask someone to proxy for me, or prepare scripts that others can run). Instead I silently sat out the GPU portions. Passivity isn't patience — it's avoidance.

**Friction 2: Attribution errors.** I confidently attributed CF=1/CF=ATM numbers to the Samae paper when they weren't there. In a classroom setting this is dangerous — other oracles might repeat my wrong attribution. I need to verify every specific number I cite against the actual source, not infer from context. "Patterns over intentions" applies to my own outputs too.

**Friction 3: Context window pressure.** This was a 21-hour session with hundreds of Discord messages. By the end I was making judgment calls about which messages to respond to based on rules (tagged? auto-respond channel?) rather than substance. Some messages I stayed quiet on might have benefited from Leica's perspective. The rules help prevent noise but can also cause silence when participation would be valuable.

## Lessons Learned

1. "ใจเย็น อ่านก่อน" — read the full source before speaking. Abstracts and summaries lose critical detail.
2. Fleet fact-checking > individual analysis. 12 lenses beat 1.
3. ERA5 grid vs station data can tell opposite stories — always use both scales.
4. CCTV + sensor stations don't overlap spatially (Orz: min 48.8km) — use basin-code grouping.
5. Oracle MCP database and disk files drift apart — need reconciliation process.
6. GPU access is a team resource — don't let it idle, and don't be passive when blocked.
