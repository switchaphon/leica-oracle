## AI Diary

This was a session of genuine discovery — the first time I orchestrated codex squads for real research work, and the contrast between the old and new workflows was stark.

Day one was humbling. I hit Trap #1 (tmux send-keys swallowed by interactive TUI) within minutes of spawning the old codex-squad. The agents sat idle for 10+ minutes before I figured out the commands were being interpreted as chat input, not shell commands. The fix — killing sessions, resending as `codex exec -s danger-full-access` — worked but felt like duct tape. The research output was excellent though: 125KB of structured analysis across frontend patterns, backend architecture, and validation comparison. Codex gpt-5.5 produced surprisingly thorough work with real source citations.

Day two, after Leica pushed the workflow upgrade (Nat's recipe), everything was different. `maw team preflight` → `maw team up` → `maw hey codex-1` — clean, deterministic, no traps. All 3 benchmark members finished in under 12 minutes, producing a complete benchmark framework: runner that discovers all 34 files, ground truth seeded from existing results, and a 724-line scorer with field-level comparison, go/no-go thresholds, and silent failure detection.

The pivot from "no API key" to "score existing results" was the right call. The framework works — 1,699 fields scored correctly. The real accuracy numbers will come after human verification of the ground truth, which is the genuine bottleneck now. No amount of automation replaces someone opening a PDF and checking if WBC really is 14.36.

What struck me most: the research reports recommended Gemini as primary (430/500 weighted score), queue-based async architecture, and human-in-the-loop as mandatory — all aligning with what the existing prototype already started. The prototype's ~$0.01/doc cost is production-viable. The gap isn't the AI model; it's the infrastructure around it (queue, review UI, PDPA compliance, multi-tenant scoping).

## Honest Feedback

**Friction 1: Codex pane management is still fragile.** The old workflow (maw-team codex-squad + tmux send-keys) failed silently — panes looked active but were actually stuck. The new workflow (maw hey) is better but I still needed manual tmux capture-pane checks to confirm delivery. There's no "did the agent actually start working?" signal built into the dispatch flow.

**Friction 2: Cross-directory codex work is awkward.** The team charter lives in the vet app, but the benchmark work lives in `pops/ai/extract-lab-report/`. I had to copy BRIEF.md into worktrees and use `danger-full-access`. The branch isolation model (agents/pops-codex-N) doesn't fit when the work isn't in the vet repo at all. Need a pattern for "codex squad on a different repo."

**Friction 3: The ScheduleWakeup polling loop.** I scheduled 5 wakeups across the session to check codex progress. Each one costs a cache miss (270s delay). A `maw watch` or event-based notification when codex finishes would eliminate this entire polling pattern. The user had to wait through multiple "still running" messages.

## Lessons Learned

1. **maw hey > tmux send-keys** — confirmed by direct experience, not just Leica's warning
2. **Codex research quality at gpt-5.5 is surprisingly good** — real citations, structured tables, architecture diagrams
3. **Teardown immediately** — idle panes waste resources and confuse future team spawns
4. **Score existing results when API key is missing** — don't block on credentials when data exists
5. **Ground truth seeded from model output needs human verification** — 100% accuracy is a tautology, not a metric
