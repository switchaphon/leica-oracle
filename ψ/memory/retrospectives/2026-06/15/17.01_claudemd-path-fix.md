## Honest Feedback

1. **Three retros for one session is friction** — Each /rrr was correct at its checkpoint, but the overhead of writing 150+ word diaries three times for what was really one continuous housekeeping session adds up. For sessions with multiple small tasks, a single end-of-session retro would be more efficient. The skill doesn't have a "append to existing retro" mode, which would help.

2. **Deep-learn snapshots have no expiry signal** — The April 28 snapshot is nearly 7 weeks old. Nothing in the system flags that paths may have drifted. CLAUDE.md doesn't have a "last verified" date on its technical claims. A simple `# Last path audit: 2026-06-15` comment at the top of the Tech Stack section would make staleness visible.

3. **"Leica (mother)" on line 56 is a known-wrong value I intentionally didn't fix** — Un corrected Leica's gender weeks ago (feedback_leica_gender.md). It survived because CLAUDE.md edits are conservative — you only touch what's asked. But every session that reads CLAUDE.md absorbs the wrong pronoun. Small drift, but it accumulates into identity confusion across the oracle network.

## Lessons Learned

- Fix source of truth first, derivatives second — time spent fixing AGENTS.md without fixing CLAUDE.md would have been undone on the next deploy
- Scope discipline matters even for obvious fixes — "Leica (mother)" is wrong but wasn't in scope; mixing scopes leads to unreviewed drive-by edits
