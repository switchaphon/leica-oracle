## Key Code Changes

The most impactful change was a 2-line addition to `~/.config/maw/maw.config.json`:
```json
"omx": "omx --yolo --direct",
"omx-resume": "omx --yolo --direct resume --last"
```

The `engineCommand()` function in `~/.maw/plugins/team/team-liveness.ts:185-188` resolves engine names via `config.commands[key] ?? config.commands.default ?? key`. Without an `omx` key, it fell back to `default: "claude"` silently.

## Honest Feedback

**Friction 1: Silent engine fallback is dangerous.** The maw engine resolution (`config.commands[key] ?? default`) silently falls back to Claude when an engine name isn't mapped. There's no warning, no log, no error. An oracle writes `engine: omx` in their charter, `maw team up` succeeds, tmux window opens, everything looks right — but it's Claude burning tokens, not omx running free. This cost real money across multiple oracles over multiple days. The fix was trivial (2 lines) but discovery required source code tracing. Maw should warn when falling back to default engine.

**Friction 2: Correction fatigue.** The codex/omx knowledge has been corrected 6+ times across 3 weeks: omx doesn't work → omx was never installed → omx works after install → engine: omx in maw still means claude → now finally fixed. Each correction is partial — it fixes one layer but misses the next. The root problem is that nobody tests the full chain. We verify the binary exists, the config passes doctor, the charter syntax is valid — but never actually spawn a team and check what engine is running.

**Friction 3: Teaching delivery is unreliable.** maw hey to rpro-ent failed multiple times across both sessions: orphan routing, wrong format, wrong pane, MCP vs filesystem inbox mismatch. Getting a message delivered and acknowledged required 3-5 attempts each time. The fallback chain (maw hey → maw send-enter → direct tmux → retry) works but shouldn't be necessary this often.

## Lessons Learned

1. **Always test the full chain** — "binary installed" ≠ "config routes to it" ≠ "maw team up launches it". Test end-to-end.
2. **Silent fallbacks are bugs** — any system that silently downgrades (omx → claude, free → paid) should warn loudly
3. **maw config location**: `~/.config/maw/maw.config.json` (not `~/.maw/`)
4. **Engine commands mapping**: `engineCommand()` resolves `config.commands[engine] ?? config.commands.default`
5. **Teaching requires verification** — send message, peek, confirm ACK. Don't assume delivered = processed.
6. **Correction cycles compound** — each partial fix creates confidence that masks the next layer's bug
