## Honest Feedback

**Three friction points this session:**

1. **Haiku agents kept hitting context limits on large codebases.** The first two architecture/API-surface agents for pawrent failed with "Prompt is too long" after reading too many files (node_modules et al). The fix was Sonnet with explicit directory exclusions. This should be the default for /learn on large repos — not a retry path.

2. **Write tool rejections slowed early file creation.** The first batch of agent files (root.md, codec.md, neon.md) was rejected, which required the user to clarify their architecture intent multiple times. A brief planning conversation before opening the Write tool would have saved 3 interruption cycles. The user's feedback was correct each time — I should have asked about naming, scoping, and architecture before writing.

3. **The awaken ritual assumes a fresh repo with no prior structure.** root-oracle already had ψ/learn/ with 18 docs. The ritual's "create ψ/ structure" step needed adaptation — I had to selectively add the missing pillars without overwriting what was already committed. This was navigable but required judgment the ritual doesn't explicitly prepare for.

## Lessons Learned

1. **For /learn on large Next.js/monorepo codebases**: Use Sonnet (not Haiku), explicitly skip `node_modules/`, `.next/`, `coverage/`, `playwright-report/`, `.git/`. Haiku hits context limits after ~30 tool uses on these projects.

2. **Agent architecture first, files second**: When setting up a multi-agent team, sketch the full hierarchy (who reports to whom, what scope, what tools/skills/plugins) and get user confirmation before writing a single file.

3. **The maw node name should match the Oracle name**: `leica` as node name creates a coherent identity across `maw ls`, `maw hey leica`, and the Oracle persona. Name consistency reduces cognitive load.

4. **Modular CLAUDE.md**: The nat-brain-oracle pattern — hub under 500 tokens + satellite detail files — is superior to a monolithic CLAUDE.md. Apply this to Leica's ~/.claude/CLAUDE.md in a future session.
