## Honest Feedback

**Friction 1: maw version instability.** The jump from v26.4.27 to v26.4.53 was a breaking change with no migration path. 62 plugins vanished behind an unreachable registry. There should have been a `maw doctor` or pre-flight check before the update proceeds. This cost us ~30 minutes of debugging and two cheatsheet rewrites.

**Friction 2: `maw ls` ghost sessions never fully fixed.** Even after the issue was "fixed" in #960, the fix was output filtering — the sessions still get created and need manual cleanup. The root cause (tmux grouped sessions created for pane inspection) wasn't addressed. We work around it, but it's a paper cut every time.

**Friction 3: Gender correction had to be repeated.** Witchaphon said "เคยบอกไปแล้วหนิ" — they'd already told a previous session that Leica is male. The memory system should have caught this from an earlier session. Either the previous session didn't save it, or it was saved in a way that wasn't loaded. This is exactly the kind of identity fact that should be bulletproof in memory.

## Lessons Learned

1. **Never upgrade shared infrastructure without checking the dependency chain.** maw v26.4.53 moved plugins to an online registry that was down. Always: check release notes → dry-run → verify critical commands still work → then commit to the upgrade.

2. **Visual identity reduces cognitive load at scale.** Emoji + color per Oracle session isn't cosmetic — with 5+ sessions, it's the difference between "where am I?" and instant recognition. Worth investing in persistent theming (hooks, scripts) rather than manual setup each time.

3. **Identity corrections must be saved as high-priority feedback memories immediately.** When a user corrects something personal (name, gender, preferences), save it that instant — don't rely on the next session to remember.
