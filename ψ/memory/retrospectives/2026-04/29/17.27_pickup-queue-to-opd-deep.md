## Architecture Decisions

1. **Prototype-first patterns over production patterns** — Sibling prototypes define the visual system. Production components have different visual weight and auth dependencies.
2. **Color tokens in consumer, shape in shared** — chips.tsx is color-agnostic. Each prototype owns its STATUS_COLOR, QUEUE_TYPE_COLOR maps.
3. **Semantic exports over type aliases** — StatusBadge and CategoryPill are structurally identical but separate exports. Callers express intent; components can diverge later.
4. **Cross-session file ownership** — page.tsx owned by queue session, opd/[id]/page.tsx by opd session. Violations caused style conflicts (OPD session overwrote HN badge styles twice).

## Honest Feedback

**Friction 1: Multi-session file conflicts are dangerous.** Two Claude sessions editing the same file without coordination caused silent regressions. The OPD session changed HN badge styles to `bg-white border-slate-200 font-bold` after we'd agreed on `bg-gray-50 border-gray-200 font-mono`. The user caught it, not us. We need a protocol: file locks, or at minimum, sessions announce which files they'll touch.

**Friction 2: Relay messages are lossy.** When the diag session sends a style alignment proposal, I parse it correctly and implement. But when the OPD session independently rewrites the same file, the alignment breaks. The relay pattern (user copies messages between sessions) works for decisions but not for atomic file operations. The extract-to-shared pattern is the real fix — single source of truth.

**Friction 3: Prototype code is still untracked.** 1,718 lines across 4 files, zero commits. The morning session's lesson "commit early, shorter sessions" was written 5 hours ago and we've been violating it the entire afternoon. The risk is real — a terminal crash or power outage would lose everything. The /rrr handoff at 14:44 was the right instinct but we should have committed the code itself, not just the retro.

## Lessons Learned

1. **Cross-session file ownership prevents conflicts** — Assign files to sessions explicitly. No cross-writing without announcing.
2. **Extract to shared is better than cross-session style alignment** — Relay messages align decisions but not file state. A shared component is the atomic fix.
3. **Prototype code must be committed incrementally** — Each milestone (flow works, filters work, shared extracted) deserves a commit. 4.5 hours untracked is unacceptable.
4. **mt-2 (8px) is the standard sub-line gap** — User tested 4px→12px→10px→8px. This is now the canonical spacing between primary and secondary content within a table cell.
