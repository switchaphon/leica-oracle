## Honest Feedback

**1. Token budget burned on git troubleshooting.** The stash/merge/branch dance consumed ~30 min of a 5-hour session. Much of it was avoidable: I should have used `git stash -u` from the start, used absolute paths consistently, and not falsely reported files as missing. These are mechanical errors, not judgment calls.

**2. UX review should happen BEFORE building, not after.** I built the prototype first, then reviewed it for UX issues, then rebuilt parts of it. If I had done the review against the Figma design before writing code, the first build would have included these improvements. Next time: review Figma → identify improvements → build once.

**3. The session is too long.** 5 hours in one context window means the early decisions are far from the current state. The handover doc helps but the context is getting heavy. Shorter sessions (2-3 hours) with clean handoffs would be more efficient.

## Lessons Learned

1. UX review the Figma design BEFORE building — not after
2. Clickable summary cards are a high-impact, low-effort pattern for any dashboard-style page
3. Status badges (color + icon + text) beat status dots (color only) per WCAG 1.4.1
4. Brand CI tokens should be absorbed early in a project, not discovered mid-prototype
5. User's domain expertise (vet practice) overrides framework recommendations — respect the rejection
