## AI Diary

This session was a lesson in humility and listening. I built the first version of the queue page with confidence — stats cards, search bar, table, modal — and it all worked. The flow was correct. But then the user said "ไม่เหมือนเล้ยยย" (doesn't look like it at all), and they were right. I had built something functional but visually inconsistent with the existing diagnostic-request-list prototype that was already established as the design pattern.

The mistake was subtle: I read the production QueueStatBox and tried to approximate the Figma layout, but the user's expectation was that all prototypes should look like siblings — same Card layout with the vertical divider and flex structure, same FilterChip rounded-full pills, same rounded-xl table with bg-gray-50/80 headers. I should have started by reading the existing prototype first, not the production code.

The filter dropdown work went much smoother. Five Figma frames told me everything: multi-select checkboxes, "เลือกทั้งหมด" toggle, optional search, count badges. The Popover+Checkbox pattern from shadcn was perfect. The user's final request to remove the filter indicator bar was a good call — it was genuinely redundant since the filter chips already show active state with count badges.

The tmux pane spawning was a detour. The user wanted another Claude session for diagnostic-request-list work. `maw wake` created a messy worktree state, and `tmux split-window` with `claude --prompt` failed silently. The positional argument `claude '<prompt>'` worked. Small lesson but worth remembering.

## Honest Feedback

**Friction 1: Not reading sibling prototypes first.** The biggest time waste was building cards/filter/table from scratch when a perfectly good pattern existed 1 directory away. I should have opened `diagnostic-request-list/page.tsx` before writing a single line. The user had to correct me, which cost ~20 minutes of rewrite. Rule: before building any new prototype, always read existing prototypes in the same directory first.

**Friction 2: maw alpha state.** `maw wake` created orphaned worktrees and ambiguous window names. The `--no-attach` flag was parsed as a worktree name. For now, raw tmux commands are more reliable than maw for spawning sessions. This isn't my fault but it cost minutes of debugging.

**Friction 3: Filter indicator bar was unnecessary.** I added it copying from diagnostic-request-list, but in that prototype it made sense because card filters were the primary filter mechanism. In this prototype, the dropdown filters already show their state via count badges. The user correctly identified the redundancy. I should evaluate whether a pattern fits the specific context rather than blindly copying every element.

## Lessons Learned

1. **Read sibling prototypes before building** — the pattern is the prototype directory, not the production code
2. **Filter chips with count badges replace filter indicator bars** — don't stack both
3. **tmux split + positional claude argument works** — `tmux split-window -h "cd /path && exec claude 'prompt'"`
