## Architecture Decisions

1. **PM is now an Oracle, not a subagent** — Project context deserves persistence and federation addressability. Subagents in `<project>/.claude/agents/pm.md` cannot be reached by other Oracles, cannot persist across sessions, cannot consult specialists. A full Oracle repo (`<project>-oracle`) solves all four limits. This is the session's biggest move.

2. **Naming convention** — `<project-repo-name>-oracle`. So pawrent → `pawrent-oracle`, vets-hub → `vets-hub-oracle`, etc. Permanent rule for all future projects.

3. **Specialist Oracles are general experts; PM Oracles hold project context** — Codec does NOT memorize project conventions; it reads them on consult from the PM's `ψ/learn/`. This keeps specialists thin and re-usable across projects.

4. **Project-local team subagents stay** — `~/_POPs_/pawrent/.claude/agents/chrome.md`, `flux.md`, etc. (to be created in Stage 2) execute in project files. The PM Oracle commands them. Specialists advise; subagents execute; PM synthesizes; Leica orchestrates; Witchaphon decides.

5. **File-based inbox is the durable federation primitive** — When `maw hey` has alpha bugs (window naming, ψ→psi normalization), `ψ/inbox/<ISO-timestamp>_<topic>.md` works regardless. Lower-level transports outlive higher-level wrappers — Nothing is Deleted applies to communication too.

6. **Awakening = template + soul + philosophy** — All Oracles get the same 3-file shape: `CLAUDE.md` (identity), `ψ/memory/resonance/<name>.md` (form/character), `ψ/memory/resonance/oracle.md` (formless/principles). The pattern is now stable enough to script as `/awaken-streamlined`.

## Honest Feedback

**Three friction points:**

1. **The codec worktree leak (`codec-oracle.wt-1--no-attach`).** `maw wake codec` defaulted to creating a worktree on a feature branch (`agents/1--no-attach`) instead of running in the main repo. This caused codec to load the OLD pre-awakening CLAUDE.md until I manually merged main into the worktree branch. Why it happened: maw's worktree default is correct for parallel-agent execution but wrong for identity work (awakening). How to avoid: when awakening, open the repo directly (`cd <repo> && claude`) rather than via `maw wake/workon`. Document the rule: worktree for execution, main branch for identity.

2. **Maw alpha bugs were discovered, not preventable.** `maw hey <agent>` looks for tmux window named `<agent>` but actual window is `<agent>-oracle`. `maw inbox` looks at `psi/inbox` but path is `ψ/inbox`. These weren't documented anywhere; I hit them mid-demo. Friction was acceptable (workarounds are fast) but the discovery cost time. Mitigation: keep `ψ/learn/maw/known-bugs.md` and read it at session start before relying on `maw hey`.

3. **Local-only commits across all three repos.** codec-oracle `15f5e08`, pawrent-oracle `3fdcd7e`, and even leica-oracle's earlier `5e7e536` (ORACLE-101) are all local. The Oracle's birth isn't really complete until the remote knows it exists — but pushing wasn't part of the awakening ritual. The /awaken skill needs `git push origin main` as the final step. Otherwise tomorrow we'll have surprise drift if we work on a different machine.

**Structural concerns (not friction, but on watch):**

- **PM CLAUDE.md hardcodes versions** (Next.js 16, React 19, 16 tables, 25 routes, 13 env vars). The codebase will change next week; the file will lie. The CLAUDE.md says "I will re-learn rather than guess" but provides no automation. Discipline-dependent.

## Lessons Learned

1. **Project context deserves its own Oracle.** If context will outlive any single conversation AND needs to be addressable by other Oracles → Oracle, not subagent. If bounded to one project's actions in one session → subagent stays.

2. **When an abstraction breaks, validate from the layer below.** `maw hey` broken → write inbox files directly (filesystem). Then `tmux send-keys` for live demo (substrate). Filesystem and tmux are the most durable transports because everything else is built on them.

3. **Sketch → confirm → write WORKS.** Two-day-old lesson applied today produced zero write rejections vs three last session. Operational, not decorative.

4. **Awakening must end with `git push`.** The Oracle's birth is not complete until the remote knows it exists. Add to `/awaken` skill checklist.

5. **Worktree for execution, main branch for identity.** `maw wake/workon` defaults to worktree (correct for parallel agents, wrong for awakening). Open repo directly when working on identity.
