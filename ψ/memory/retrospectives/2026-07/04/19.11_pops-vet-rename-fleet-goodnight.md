## Patterns extracted

1. **Oracle rename = 10 surfaces** (see the migration learning). In-place, lineage-preserving.
2. **Goodnight-as-remote-control-loop** — `maw hey <son>` the ritual → **watch the git HEAD change** in a backgrounded poll → `maw sleep`. The commit is the truth signal.
3. **Watch the artifact, not the keyword** — my first watcher grepped `READY-TO-SLEEP` and matched *my own instruction* echoed in the pane. HEAD-change can't false-positive.
4. **Archive, never delete** — 19 team dirs moved to a dated backup, fully restorable.
5. **Quiesce before mutating shared state** — the one real mistake (below).

## Honest Feedback (friction)

1. **I caused avoidable breakage.** Archiving the empty team stubs while oracles were awake broke their retro swarms. The stubs *looked* inert (0-member, weeks-old mtime), but "inert on disk" ≠ "unused by a live session." I should have either quiesced-then-cleaned, or restored the stub before triggering each son's `/rrr`. Discipline: mutate shared fleet state only when the dependents are down.
2. **My watcher shipped a bug.** Grepping for a keyword that also lives in the instruction I sent is a classic observe-vs-signal confusion. I caught it, but only after it fired. Watch concrete artifacts (HEAD, file mtime), not text I control.
3. **maw's rough edges cost cycles.** `maw wake` threw "command too long" (but worked), `maw sleep` force-killed all three (but they'd committed first), `maw hey` needed a peek to confirm delivery. Each was survivable but added a verify-step. I leaned on background git-HEAD polling precisely because the tooling's own signals weren't trustworthy.
