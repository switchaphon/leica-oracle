## AI Diary

This was a marathon session. What started as "build a drawer" turned into a deep crawl through 8+ files to understand the existing patterns — QuickViewDrawer's timeline, LabTestSelector's right pane, production LabOrderDrawer's lifecycle, diagnostic store's shared state. Every time I thought I was done, Un would spot something that needed alignment: the blue info card was redundant (they were right — all that data was already in the header), the timeline needed duration like the queue drawer, "ดำเนินการ" was breaking the time column layout.

The hooks order violation was a genuine mistake — I put `useState` after an early return in PrototypeGuide, which is a fundamental React rule I should never break. That caused the open-quick-view page to crash entirely. Embarrassing, but fixed quickly once the error was clear.

The most satisfying part was the iterative refinement cycle. Un kept pushing for consistency: same timeline pattern across queue and diagnostic drawers, status chips instead of generic text, sidebar active state matching prototype routes. Each iteration made the whole prototype system more coherent. The PrototypeSideBar was a great call — it decouples prototype routing from production entirely.

I'm still not fully happy with the guide beacon approach for the multi-case review page. 7 beacons was too many, 1 is clean but loses discoverability. The legend panel with case descriptions helps, but I wonder if a dedicated "case switcher" UI inside the drawer itself would be better for review flows.

## Honest Feedback

1. **Hooks placement**: I should always structure React components with ALL hooks at the top before any early returns. The crash from `useState` after `if (!mounted) return null` was avoidable — it's React 101. Need a mental checklist before writing component bodies.

2. **Logo path guessing**: I assumed the logo import path from memory instead of checking the actual import in SideBar.tsx first. Cost an extra round-trip. Always `grep` the existing import before writing a new one.

3. **PAGE_SIZE change was heavy-handed**: Changing from 5 to 15 rows to make all cases visible on one page affects the default diagnostic page too, not just open-quick-view. Should have made it a prop or kept the pagination and found another way to guide reviewers to page 2.

## Lessons Learned

- Always place ALL React hooks before any early return — hook order must be identical across renders
- When adding patterns to a drawer, check all existing drawer usages and apply consistently (timeline chip applied to 3 QuickViewDrawer sites + DiagnosticQuickViewDrawer)
- DS + prototype index update is not optional — it's part of "done" for any prototype work
