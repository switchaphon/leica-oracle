## Lessons Learned

- **Gate only the expensive, irreversible direction** (asymmetric friction). ล่วงหน้า spawns an appointment → gate it; เพิ่ม is cheap/correctable → no gate.
- **Reuse a DS pattern for mode differentiation** before inventing one — the vet-schedule modal's full-width tab became the order-mode selector; the active tab is its own title.
- **An entity's coupling field ≠ "an entity of that kind exists nearby"** — a เพิ่ม order's `appointment_no` stays NULL even when a follow-up appointment coexists in the same visit; they're independent objects.
- **At the start of a design grill, read the brain's recent learnings/audits**, not only the code. (See lesson file.)
