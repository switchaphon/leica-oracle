## AI Diary

This was a satisfying cleanup session. I picked up from a context compaction — the previous session had been a marathon of X-ray body map implementation with dozens of screenshot-driven iterations. Starting from a summary meant I had to carefully re-read the exact state of the code rather than relying on conversation memory.

The extremity delete crash was a classic React anti-pattern that I should have caught the first time: calling `toggleRegion` (a state setter) inside a `setExtSites` updater function. In strict mode, React can re-invoke the updater, and by that point the site had already been deleted from the previous invocation's return. Moving `toggleRegion` outside with a `willBeEmpty` flag was the clean fix. I'm noting this pattern for future — never put cross-state side effects inside updater callbacks.

The grouped summary pane work was the most impactful change. Three modals, all with flat item lists, now have proper category headers. It makes the right pane actually scannable. I delegated Lab and Ultrasound to sub-agents in parallel while doing X-Ray myself — good use of parallelism since all three files are independent.

Today I finally aligned it to the actual dashboard source code (`text-brand + RotateCcw`), not my imagination of what it should look like. The lesson is clear: always grep the live codebase for the pattern before inventing one.

**1. Context compaction recovery is fragile.** The summary captured the right bugs and pending tasks, but I lost the nuance of *why* certain design decisions were made. I had to re-read files to reconstruct context that was free in the previous turn. For long design-iteration sessions, a mid-session checkpoint (like a HANDOFF.md) would preserve more than the automatic compaction summary.

**2. The reset button should have been right the first time.** This is the third session touching the same button. The pattern was already in the codebase (`dashboard/page.tsx` line 560) — I should have grepped for `รีเซ็ต` or `RotateCcw` before ever writing a new style. The feedback memory "audit live before DS mock" applies here and I still didn't follow it instinctively.

**3. Post-commit discipline was missing until Un reminded me.** DESIGN.md and changelog updates should be reflexive — they're part of the commit, not separate tasks. The fact that Un had to ask means the previous sessions' commits were shipped without updated exports. This is exactly the kind of drift that makes DS unreliable. Now saved to memory.

## Lessons Learned

1. **Never call cross-state setters inside React state updater callbacks.** Use a closure variable (`willBeEmpty`) to communicate intent out, then call the second setter after the first returns.
2. **Post-commit checklist is non-negotiable:** (a) re-export DESIGN.md, (b) update overall + per-flow changelogs.
3. **When matching an existing pattern, grep the codebase first.** Don't design from memory — `grep -rn 'รีเซ็ต' src/` would have given the exact style in 2 seconds.
