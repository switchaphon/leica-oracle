## Honest Feedback

**1. Parallel session coordination is ad-hoc.** I found files already modified by another session, but had no signal about what was done or why. The "file has been modified since read" error is the discovery mechanism — not great. A simple convention like writing a `.wip` marker or updating a shared state file would prevent wasted analysis.

**2. Naming inconsistency should have been caught at creation time.** DISPENSING vs WAITING_MEDICATION for the same concept — this existed since the dashboard mock was created. A grep for the concept "รอจ่ายยา" across both mocks would have revealed it instantly. Convention: all "waiting for X" statuses should follow `WAITING_*` pattern.

**3. Post-commit checklist still not automatic.** I saved the rule, committed, and didn't do the DS export or changelog. The rule is in memory but not in muscle. Need to find a way to make this a blocking step — maybe a pre-push hook or a checklist in the commit message template.

## Lessons Learned

1. **WAITING_* naming convention for queue statuses.** All "waiting for X" statuses should use `WAITING_` prefix: WAITING_LAB, WAITING_PAYMENT, WAITING_DISPENSING. Makes grep and type narrowing easier.
2. **Extend shared components with className, not new components.** When a shared component needs layout variation in one context, pass className rather than forking.
