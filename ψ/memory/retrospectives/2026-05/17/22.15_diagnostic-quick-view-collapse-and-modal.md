## AI Diary

This was one of the most iterative UI sessions I've had on the vet project. Un started with a clear problem — the diagnostic drawer scrolls too long when there are many test orders — and we used grill-me to systematically walk through every design decision before writing a single line of code. Nine questions, each building on the previous answer. That discipline paid off: when I started coding, I had zero ambiguity about what to build.

The modal went through at least six layout iterations. Each time Un saw the screenshot, they spotted something — the right column is taller than the left, the X button isn't aligned, the close button should be centered not full-width, move this here, put that there. What struck me is that none of these were arbitrary — each adjustment came from a real UX instinct about what a lab technician or receptionist would need when reading a work order under pressure.

The species icon removal was a good example of a change that started small ("just in the drawer") and correctly expanded to "everywhere except tables." Un knew exactly where the boundary was — tables need icons for quick scanning, but headers and modals don't. The Thai text ("สุนัข · Siberian Husky · เพศผู้") is actually more information-dense than a Dog icon that only conveys one dimension.

The modal size tokens feel like they'll compound. Every future modal debate can now start with "is this S, M, L, or XL?" instead of picking arbitrary pixel values.

## Honest Feedback

**Friction 1: Playwright screenshot reliability.** At least 3 screenshot attempts failed with timeouts or missing files during this session. The dev server was running (curl returned 200) but Playwright couldn't connect. I wasted cycles retrying with different timeout values and waitUntil strategies. This should be a solved problem — a robust screenshot helper script that handles HMR delays.

**Friction 2: Cascading species icon changes.** When Un said "ทุกที่ยกเว้นตาราง," I had to hunt through 12 files to find every SpeciesIcon usage. There's no single source of truth for "how we display pet identity" — it's inline JSX scattered across pages. The PatientHeader shared component helps, but several pages (queue/call-to-opd, pet/page) render their own pet headers outside of it. A shared `PetIdentityLine` component would prevent this next time.

**Friction 3: Iterative positioning feedback loop.** Each layout tweak (move X button, move order number, move date, change close button width) required a full cycle: edit → build → screenshot → review → edit again. Six cycles for the modal header alone. This is inherent to visual work, but I wonder if showing Un a quick ASCII mockup before coding would have collapsed some of these rounds.

## Lessons Learned

- Collapsing detail into timeline cards (queue drawer pattern) is a proven pattern for reducing scroll — apply it whenever a drawer section is purely informational
- Modal size tokens should be defined early in a design system; ad-hoc max-w-* creates inconsistency
- Species text > species icon in read-heavy contexts (work orders, headers) — icons work in scan-heavy contexts (tables)
