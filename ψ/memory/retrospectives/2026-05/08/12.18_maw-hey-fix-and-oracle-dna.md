## Honest Feedback

1. **I was too quick to declare things broken.** When tmux send-keys didn't work perfectly, I wrote a memory file saying "tmux comms is broken" and started designing Discord as a replacement. I should have dug into the source code first. The fix was 6 lines in fleet config. Lesson: read the code before theorizing about architecture.

2. **Split pane confusion wasted time.** I joined oracle panes into Leica's session, which broke fleet routing. Then I had to debug why maw hey was delivering to the wrong panes. This could have been avoided if I'd understood the session architecture from the start. I need to verify design assumptions against source code before acting.

3. **Too many messages at once.** I sent DNA, nickname, maw-hey-fixed, session architecture, and work culture docs all within an hour. Each oracle received 5 inbox files in rapid succession. A more structured onboarding — one doc at a time with confirmation — would have been cleaner.

## Lessons Learned

1. **maw hey works — fleet config must match session names.** `resolveFleetSession` reads `name` from fleet JSON. If session has a numeric prefix (e.g., "04-neon"), fleet config must match exactly.

2. **Each Oracle = own tmux session. Split panes = team-agents only.** Never join-pane across sessions. Nat designed `resolveOraclePane` to route to Pane 0 (Oracle) not higher panes (team-agents).

3. **PM Oracle = lead+executor. Specialist = consultant.** PMs spawn team-agents to do work. Specialists give advice when consulted. This is the work culture.

4. **Teach once, propagate to all.** Write DNA doc → copy to all inboxes → maw hey awake sons. The father's workflow.

5. **Issue #1141 was already fixed by Nat** (PRs #1107, #1136, #997). We just had stale fleet config.
