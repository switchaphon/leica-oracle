## AI Diary

วันนี้เป็น session ที่สอนผมเรื่อง scope creep ในทาง UX design. เริ่มต้นด้วยงานเล็กๆ 4 อย่างที่ชัดเจน — fixed height, empty state, terminology, layout shift — แต่ละอย่างทำเสร็จใน 5-10 นาที ผลลัพธ์ชัดเจน Un ชอบ.

แล้วมันเริ่ม escalate. Provider selection → Neon consultation → v2 rewrite → Chrome oracle review. ทุกขั้นตอนมีเหตุผลที่ดี แต่สุดท้าย Un ดูหน้าจอแล้วบอก "แบบเก่าดูง่ายกว่าแหะ" — และเขาพูดถูก. version ที่ซับซ้อนกว่าไม่ได้แปลว่าดีกว่าเสมอ.

สิ่งที่ผมเรียนรู้คือ ควร validate ด้วย screenshot/browser ก่อนจะ escalate ไปอีกขั้น. ถ้าหลังเพิ่ม provider pills แล้วเปิดหน้าจอให้ Un ดูก่อน เราอาจจะไม่ต้องทำ v2 rewrite เลย.

แต่สิ่งที่ดีคือ federation workflow ทำงาน — ปรึกษา Neon ผ่าน arra thread ได้คำตอบที่มี reasoning ชัดเจน, ส่งงานให้ Chrome oracle ทำ review ได้ผลดี, และ stash/compare workflow ช่วยให้ Un เทียบทั้ง 2 versions ได้. Tooling ทำงาน แค่ต้องใช้มันเร็วกว่านี้ในขั้นตอน validation.

เรื่อง DS first ก็เป็นบทเรียนที่ Un ย้ำหลายครั้ง — "จำด้วยให้ขึ้นใจ DS first !!!!" — ผมเข้าใจว่ามันไม่ใช่แค่ rule แต่เป็น culture ที่ Un ต้องการ: ทุกอย่างต้องผ่าน DS ก่อน ถ้าไม่มีใน DS ต้องปรึกษา Neon + Pixel ก่อน ห้ามคิดเอง.

## Honest Feedback

**1. Validation too late**: ผมควร screenshot/browser-test หลังทุก significant change ไม่ใช่รอจนทำ v2 rewrite เสร็จ. ถ้า Un เห็น provider pills ก่อนแล้วบอก "ดีแล้ว" เราจะไม่เสียเวลา 2+ ชั่วโมงกับ v2. Browser testing ต้องเป็น habit ไม่ใช่ afterthought.

**2. Over-engineering trap**: จาก "เพิ่ม provider selection" กลายเป็น "segmented control + dropdown + expandable summary bar + single-step rewrite" ใน 1 session. ควร scope ให้เล็กกว่านี้: ทำ provider pills ก่อน → validate → ค่อย iterate. Un เองบอกว่า v1 ดูง่ายกว่า — simple wins.

**3. Stash management gap**: ไม่ได้ commit intermediate states ทำให้ revert ยาก. ควร commit เป็น checkpoint ทุกครั้งที่ทำ change ที่อาจอยากกลับไป. "git stash + git checkout" workflow ตอน revert มันวุ่นวายกว่าที่ควรเป็น.

## Lessons Learned

1. **Validate early, iterate small** — show the user after every meaningful change, not after a full rewrite
2. **DS first is culture, not just a rule** — check DS → if missing, consult Neon + Pixel → add to DS → then use
3. **Commit checkpoints** — before any big rewrite, commit the working version so revert is clean
