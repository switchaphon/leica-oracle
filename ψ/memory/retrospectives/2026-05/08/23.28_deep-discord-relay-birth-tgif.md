## Honest Feedback

**1. Discord plugin debugging was wasteful.** I spent 90 minutes on MCP reconnects, process checks, and config file shuffling when the root cause was un-enabled Privileged Intents — a 10-second fix in the Discord Developer Portal. The standalone test script (`/tmp/test-discord-bot.ts`) that proved the real error should have been my FIRST step, not my last. Patterns Over Intentions means test the actual thing, not the surrounding infrastructure.

**2. tmux send-keys is unreliable for inter-agent communication.** I kept using it despite knowing from today's learnings (and documented in the DNA!) that it doesn't work well. Messages got concatenated, commands got stuck, and the relay session ended up running commands meant for the shell inside Claude as prompts. `maw hey` works better but still has edge cases. Discord (ironically) is the solution I was trying to build.

**3. Overloaded the session.** 5 hours, 4 major activities (multi-account research, Discord setup, relay bud, relay awaken), 3 blockers debugged, 1 oracle born. Each of these deserved its own focused session. The "ไปให้จบ ๆ" energy is motivating but exhausting — both for token budget and for clean retrospective writing. Un's instinct to ship is right; the execution should be more sequential.

## Lessons Learned

1. **Always test the actual service first** — standalone discord.js script found the Intents error in 3 seconds. MCP debugging found nothing in 90 minutes
2. **`start.sh` must run FROM SHELL, not inside Claude** — `exec claude` replaces the process; it can't be a prompt inside an existing session
3. **Auto mode blocks MCP tools by default** — Discord reply/react/edit need explicit permission in `.claude/settings.json`
4. **Privileged Intents are required** — Discord bots using discord.js need all 3 intents enabled even if code only uses MessageContent
5. **Bud must register in fleet** — `maw wake` after bud, verify in `maw ls` under Fleet not Other
6. **The Circuit is real** — not a plan, not a design doc. A live Discord server with a connected bot that responds
