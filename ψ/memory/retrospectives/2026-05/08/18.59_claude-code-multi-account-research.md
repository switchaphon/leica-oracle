## AI Diary

This was a satisfying session in its simplicity. Un came in with a specific question — can I use these proxy tools to switch between 2 Claude Code Max accounts? — and within 23 minutes we had a clear answer: no, those tools solve a different problem, and the native approach is both safer and simpler.

What I appreciated was Un's instinct when we got to claude-swap. I presented it as an option with caveats, and he immediately said "risky" (เสี่ยงแหะ). That's the right call. Giving a third-party tool access to your OAuth tokens is a real trust decision, and for something you do infrequently, the 10-second native logout/login is the obvious choice.

The research itself was instructive. The Claude Code proxy ecosystem is solving for cost optimization (routing to cheaper models) and enterprise governance (agentgateway), not for the simpler use case of "I have 2 subscriptions." The fact that Max uses OAuth rather than API keys creates a natural barrier that most proxy architectures can't bridge cleanly. This is actually a good security property — it means your subscription can't be trivially proxied or shared.

I also noticed the gap: Anthropic doesn't have native multi-profile support yet (issue #44687). For someone running an Oracle fleet across multiple accounts, this will matter more over time. But for now, Un's use case is simple enough that native commands suffice.

**3. The CCS recommendation was premature.** I recommended CCS for parallel use without confirming it actually works reliably with Max OAuth. Scout found the tool but didn't verify hands-on. Recommending tools I haven't tested goes against patterns-over-intentions.

## Lessons Learned

- Claude Code Max accounts use OAuth (macOS Keychain), not API keys — proxy tools expecting API keys don't work with Max
- `.claude/` folder (settings, skills, memory) is machine-scoped, not account-scoped — switching accounts keeps all config
- Native `claude auth logout` + `claude auth login` is the safest multi-account approach for infrequent switching
- Third-party token managers (claude-swap) store OAuth tokens in plaintext — unnecessary risk for simple use cases
