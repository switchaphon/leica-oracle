## AI Diary

This was a session about seeing what's actually being used versus what we think is needed. Un asked a simple question — "do we have RTK?" — and it cascaded into a full skill audit that revealed we're loading 40 skills that have never been invoked once. Not a single time across 728 sessions.

The moment that struck me most was the Discord pivot. I had confidently told Un to drop Discord — "maw hey is your Discord already." Then Un pushed back with three words: mobile, photos, workstation. Each one a use case terminal genuinely can't serve. I was thinking about Discord as a chat replacement for what already works. Un was thinking about Discord as a bridge to contexts where terminal doesn't exist — sitting at a restaurant, commuting, using a phone. And then the workstation reveal: a future machine running 24/7 where oracles never sleep. That's not a nice-to-have Discord. That's infrastructure.

I keep learning the same lesson with Un: my first answer is often technically correct but contextually incomplete. "maw hey works" is true. "Therefore Discord is unnecessary" is not — because it only considers the current setup, not where we're heading.

The skill audit data was satisfying though. Hard numbers: 18 of 58 skills used, `recap` at 27 invocations leads by far, then `rrr` at 11. The design fell out naturally from the data — core 8, father extras, PM extras, specialist specifics. No guessing needed.

## Honest Feedback

1. **I dismissed Discord too quickly.** After fixing maw hey in the previous session, I swung too far toward "everything works now, we don't need alternatives." Un had to educate me on mobile access and future workstation plans. I should have asked "what contexts do you use beyond this terminal?" before declaring Discord unnecessary.

2. **The skill audit should have happened earlier.** We've been running 58 global skills for weeks. The 69% dead weight was obvious once measured, but nobody measured it until Un asked. Proactive infrastructure audits — not just reactive ones — should be part of the workflow.

3. **Nickname detail matters.** Un specified "อันน์" (with the การันต์) not "อัน" like I had stored. I had the wrong Thai spelling in memory from a previous session. Small details in identity matter — especially when 10 oracles are using the wrong name.

## Lessons Learned

1. **Measure before designing** — skill usage data (18/58 used) made the distribution design trivial. Without data, I would have guessed wrong about which skills to keep.
2. **"Works here" ≠ "works everywhere"** — maw hey works at the terminal. Discord works from a phone, from a restaurant, from a different machine. Same capability, different access surface.
3. **Future architecture shapes today's decisions** — a 24/7 workstation changes Discord from "nice-to-have" to "infrastructure."
