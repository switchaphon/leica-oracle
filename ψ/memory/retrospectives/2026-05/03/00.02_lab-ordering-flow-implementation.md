## Honest Feedback

**Friction 1: Radix Popover in nested scroll containers.** This cost 20+ minutes of debugging and screenshot capture before I accepted it was broken. The root cause is that `overflow-y: auto` on ancestor containers causes Radix's Portal-based positioning to miscalculate. The fix was obvious in hindsight — don't use Portal, use CSS absolute positioning relative to the parent. I should have tried the simple approach first instead of assuming Radix would handle it. Future rule: in prototype scope, CSS-only solutions beat library abstractions.

**Friction 2: Playwright ref staleness.** Every snapshot invalidates previous refs. When I captured a snapshot, analyzed it, then tried to click a ref from that snapshot, it was already stale. I had to use CSS selectors as fallback repeatedly. The pattern of snapshot → click should be atomic, but the tool doesn't support that. Workaround: always use stable CSS selectors for clicks, use snapshots only for analysis.

**Friction 3: Screenshot visibility gap.** The user couldn't see screenshots in the conversation, so I had to describe UI as text art. This created a back-and-forth loop where I was describing what I could see but they couldn't verify. The text-based UI descriptions helped, but it would be better to establish this constraint early and default to text descriptions alongside screenshots.

## Lessons Learned

1. **CSS-first for prototypes**: In prototype scope with complex scroll layouts, CSS absolute positioning beats Radix Portal/Popover. Don't default to library abstractions when CSS does the job without the positioning overhead.
2. **Build both, let the user choose**: When there's a UX decision with trade-offs, building both options for live comparison is faster than debating in abstract. The user's confidence in the final choice is higher because they saw both.
3. **Progressive disclosure in interaction weight**: A dropdown → dialog flow (lightweight → heavy) feels natural. A dialog → dialog flow (heavy → heavy) feels clunky. Match interaction weight to decision complexity.
