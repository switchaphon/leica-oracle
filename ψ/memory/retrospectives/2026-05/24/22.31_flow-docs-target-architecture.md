## Key Architecture Decisions

1. **OPD has its own lifecycle** — IN_PROGRESS → PAUSED → WAITING_DIAGNOSTIC → COMPLETED (separate from Queue)
2. **Queue uses generic IN_SERVICE** — wrapper state that delegates detail to child entities (OPD, Grooming, etc.)
3. **WAITING_LAB → WAITING_DIAGNOSTIC** — renamed and moved from Queue to OPD level, covers all modalities
4. **wait_for_result is OPD's concept** — determined by SOAP section: Objective = wait, Plan = no wait
5. **State vs Status pattern** — uniformly applied (lifecycle vs soft-delete) across all 3 entities
6. **Target over As-Is** — all docs now document the proposed architecture, not current implementation

## Honest Feedback

**Friction 1: Vercel plugin noise** — Every Read/prompt triggered Vercel skill injection suggestions (next-cache-components, nextjs, workflow, vercel-sandbox, ai-sdk) that were completely irrelevant to writing HTML flow documentation. These injections consumed context and added cognitive overhead. The pattern matching is too aggressive for non-code files.

**Friction 2: Static HTML in Next.js routing** — The prototype index page uses Next.js `Link` for navigation, but flow docs are static `.html` files that can't be routed through App Router. Had to add conditional rendering (`<a>` for flows, `<Link>` for everything else). This is a friction point in the architecture — flow docs live in the App Router directory but aren't React components.

**Friction 3: Agent context isolation** — Each subagent had to re-read both template files (appointment-flow + queue-flow) from scratch because they don't share context with the parent. For 3 agents reading the same 2 files, that's 6 redundant file reads. A shared context mechanism would be more efficient.

## Lessons Learned

1. **Grill before build** — 45 minutes of structured questioning saved hours of rework. The ratio of planning-to-execution was 3:1 and it felt right.
2. **Target architecture docs are specs, not history** — Documenting As-Is creates throwaway work. Document what you're building toward.
3. **State ownership clarity matters** — Queue owns the generic pipeline; OPD owns clinical detail. This separation was fuzzy until the grill forced explicit decisions.
4. **wait_for_result = clinical intent encoded as system behavior** — Objective (need results now) vs Plan (results for next visit) is a domain insight that maps cleanly to a boolean flag.
