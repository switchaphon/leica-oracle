## AI Diary

This was the longest single-day session I've had on the OPD prototype. Nine and a half hours, three commits, nine files, and three hard lessons. The day started with Figma mockups and ended with a deep retro synthesized by five parallel agents.

The most important thing that happened today wasn't any individual feature — it was the establishment of a boundary. "ห้ามไปแตะต้องของอย่างอื่นใน production เลยเด็ดขาด" — Un said it once, clearly, and I will never cross it again. Every prototype change lives in `/prototype`. If it needs to override shared CSS, create a local file. If it needs a different component, wrap or copy. Never modify the source.

The second most important thing was learning to stop arguing from code when the user is arguing from pixels. "The CSS is 12px in both sections" was technically correct and practically useless. Un sees the screen, not the stylesheet. The min-height gap illusion taught me that visual truth and structural truth can disagree — and in UI work, visual truth wins.

The AI Health Snapshot was satisfying to build because it represents a genuine product insight: a vet opening a case needs 5-7 sentences of context, not raw visit records. Active problems, current meds, allergies, weight trend, vaccination status — that's a 5-second scan that prevents mistakes. The old history cards were data; the snapshot is information.

I'm proud of the spacing system too. `mt-3 space-y-2` as a wrapper pattern is simple, teachable, and consistent. It took an hour of debugging to arrive at something that fits in one line of CSS.

## Lessons Learned

1. Prototype isolation is absolute — never modify production files
2. min-height creates visual gap illusion — measure pixels, not CSS
3. AI attribution in medical UI belongs at footer as citation
4. Stock bundles (ตัดสต็อกคู่) are inventory management, not pricing
5. Visual truth > structural truth in UI work
