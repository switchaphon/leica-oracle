## Honest Feedback (friction points)

1. **Explore subagents failed early** ("Prompt is too long") — in a long/loaded session the Agent tool's combined context exceeded a limit, so I lost parallel fan-out and did all exploration inline. It worked but cost serial round-trips; worth knowing that subagents are unreliable once the main context is heavy.
2. **Figma bridge wasn't connected** — the first two `figma_get_status` probes failed because the Desktop Bridge plugin wasn't open; it took an explicit ask to Un and a reopen. The visual-fidelity questions were blocked until then, so I front-loaded the codebase-answerable exploration to not idle.
3. **Session length vs context economy** — 600K in one session is a lot; the prompt-cache breakpoints and re-reads add up. I should have proactively proposed the handover around the 9-flows milestone instead of continuing into the pet-profile build, letting Un decide the cut point.

## Lessons Learned

- **A feature's "new axis" may already exist** — grep/read for it before designing storage or structure; the cheapest spine forwards an existing signal (here, `section` → `DiagnosticContext`).
- **Verify prototype UX with Playwright CLI click-tests**, not just `tsc` — compilation ≠ behaviour, and it catches library quirks (Radix `asChild`).
- **DS-first = reuse the composite, enhance it backward-compatibly, build+flag the gaps** — `PatientHeader` got a conditional `เปิดแฟ้ม`; `OwnerCard`/`PetProfileOverview` are new + flagged for the DS page.
