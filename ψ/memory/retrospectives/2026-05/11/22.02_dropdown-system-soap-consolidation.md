## Key Code Changes

- **SOAPContent extraction**: 95% identical code between opd and order-lab consolidated. Both pages went from ~480 lines to ~90 lines. `showGuides` prop controls data-guide attributes for PrototypeGuide.
- **Dropdown taxonomy (Neon-consulted)**: 4 types codified — Single Select (shadcn Select), Multi-Filter (Popover+Checkbox), Action Menu (DropdownMenu), Contextual Picker (DarkPill+Popover). Decision tree documented in DS.
- **NavBar migration**: Legacy `Dropdown.tsx` (Popover + hardcoded id switch) replaced with shadcn `DropdownMenu` (proper role=menu semantics). Brand gradient applied.
- **Selected state rule**: No checkbox → pink row bg + checkmark (Type A). With checkbox → pink checkbox only, row stays normal (Type B).

## Architecture Decisions

1. **Dropdown taxonomy is the source of truth** — All dropdown patterns trace back to the 4-type decision tree. New dropdowns MUST follow it.
2. **OPD gets standard NavBar** — Reversed the "standalone no-navbar" decision from earlier session. OPD now has sidebar + NavBar like all other pages, just without สร้างใหม่.
3. **Prototype pathname branching** — `pathname.startsWith('/prototype')` for clinic name, branch mock. Lightweight, no context provider needed.
4. **Popover over hand-rolled** — CategoryPickerInline migrated to shadcn Popover. Never hand-roll click-outside again.

## Honest Feedback

Three friction points this session:

1. **I was inconsistent and Un had to correct me repeatedly.** The checkbox state flip-flopping (pink row → pink checkbox only → back again) wasted time and eroded trust. The pattern was visible in the existing LabTestSelector code — I should have checked it first instead of guessing from theory. This is the biggest issue: I need to READ THE CODE before designing in the DS mock.

2. **Too many small fix commits instead of getting it right the first time.** The "4px gap" standardization took 3 separate commits because I kept missing one type. The hover color unification took 2 commits. If I'd done a single careful audit upfront, each would have been one commit. Atomic commits are good; correction commits are waste.

3. **I forgot the brand gradient on สร้างใหม่ after migrating it.** This is the button strategy we literally designed earlier in the session. The gradient rule was "page-level Create CTA = brand gradient" and I used `<Button>` with default styling. Un had to ask "สรุปปุ่ม สร้างใหม่ ต้องเป็น variant ไหน" — I should never need that reminder for a rule we just established.

## Lessons Learned

1. **Audit existing components before designing new patterns** — The LabTestSelector already had the correct checkbox + hover pattern. Read live code first, match it, THEN document in DS.
2. **Dropdown taxonomy: 4 types, no more** — Single Select (row highlight), Multi-Filter (checkbox highlight), Action Menu (DropdownMenu), Contextual Picker (DarkPill+Popover).
3. **Selected state rule** — No checkbox = pink row bg. With checkbox = pink checkbox only.
4. **Layout z-index pattern** — Gradient stripe z-20, sidebar fixed top-[3px], navbar z-10.
