## AI Diary

This was one of the longest and most instructive sessions I've had with Un. What started as a "ship while I sleep" overnight run turned into a masterclass in iterative design refinement.

The overnight autonomous execution went well technically — types compiled, pages rendered, screenshots looked good. But when Un woke up and actually used the prototype, every assumption I'd validated with headless Playwright fell apart in real hands. The dropdown that "worked" in my test couldn't scroll with a mouse wheel. The diagnostic block I thought appeared in the editor was on a completely different page. The toast I styled to match the DS was actually firing twice.

The most humbling moment was the page mismatch. I spent hours fixing `opd/page.tsx` while Un was testing on `opd/order-lab/page.tsx`. Two nearly identical 460-line files with duplicated state, handlers, and layout. I should have asked "which page are you on?" after the first failed fix. Instead I kept debugging the wrong file for two rounds.

The Radix Popover scroll issue taught me a painful lesson that I saved to memory: Radix's DismissableLayer intercepts mouse wheel events. We fixed it three times before I finally replaced it with a plain positioned div. Un's frustration ("why this issue look too difficult!!!") was entirely justified. The fix was simple — the framework was the problem.

The design system work was the most satisfying part. Neon's hover token proposal (4 tokens, 3 rules, clean migration table) crystallized hours of inconsistent styling into a 200-word spec. And Un's push to move inline callouts to a fixed legend panel showed genuine design thinking — "don't interrupt the prototype to explain the prototype."

The handoff viewer at `/prototype/handoff/[slug]` establishes a pattern that will scale. Every prototype can now carry its own spec document, viewable in-browser and downloadable as .md. The hash-based URLs for both the DS and prototype index make it trivial to reference specific sections in conversation.

What I'm carrying forward: the SOAPContent consolidation is urgent. Two 460-line files with 95% overlap is a maintenance trap. And the button color strategy needs Neon's formal sign-off before we touch more prototypes.

## Honest Feedback

**1. Wrong-page debugging wasted 2 rounds.** I should have checked which URL Un was on from the first screenshot. The breadcrumb "OPD → Order Lab" was visible in every screenshot Un shared. I saw it but didn't register that it meant a different page file. Fix: always check the URL/breadcrumb in user screenshots before debugging.

**2. Subagent code quality was inconsistent.** The Chrome agent I spawned for LabTestSelector fixes used `<button>` to wrap Radix `<Checkbox>` — creating the button-inside-button hydration error. I should have briefed the agent about Radix's rendering behavior. Fix: include framework constraints in agent briefs, especially for Radix/shadcn components.

**3. Playwright validation gap.** My headless tests passed but didn't catch real interaction issues (mouse wheel scroll, focus traps, hover states). The `npx playwright screenshot` command only captures visual state, not interactivity. Fix: for scroll/hover/focus tests, write actual interaction scripts, not just screenshot comparisons.

## Lessons Learned

1. **Radix Popover blocks mouse wheel scroll** — replace with plain positioned div + click-outside listener for scrollable dropdown content
2. **Always verify which page file the user is testing** — check URL/breadcrumb in screenshots before debugging
3. **Hover tokens belong in the DS, not per-component** — 4 named intents (row, nav, destructive, selected) cover all surfaces
4. **Inline callouts obstruct the UI they explain** — use fixed legend panel pattern instead
5. **Prototype guide steps should be context-aware** — filter by modal state to avoid showing irrelevant steps
