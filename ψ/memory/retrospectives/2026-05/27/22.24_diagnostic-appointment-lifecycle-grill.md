## Key Decisions Reached

1. **`source_visit_no`/`target_visit_no` vs `appointment_no` serve different purposes** — visit_no = data routing (SOAP attachment), appointment_no = lifecycle coupling (cascade state changes). Both needed.
2. **Case B (advance order without appointment) must not exist** — UX must prevent or strongly discourage this.
3. **Two valid edge cases identified**:
   - Edge 1: Vet creates "สั่งตรวจล่วงหน้า" but forgets appointment → UX guardrail needed
   - Edge 2: Appointment exists, vet adds diagnostic order later (phone call) → `source_opd_id NOT NULL` constraint blocks this
4. **Proposed Option C for v1** (verbal handoff for phone-call scenario), design toward Option B later ("pre-visit note" concept) — Un hasn't confirmed yet.
