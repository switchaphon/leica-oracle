## Architecture Decisions

1. **5 appointment states, no COMPLETED** — BOOKED, CHECKED_IN, CANCELLED, NO_SHOW, RESCHEDULED. Lifecycle ends at CHECKED_IN; Queue takes over.
2. **RESCHEDULED = create new + mark old** — bidirectional chain (reschedule_to/from), not an edit. Supports multi-hop A→B→C.
3. **State ≠ Status orthogonality** — Cancel changes state (BOOKED→CANCELLED), Delete changes status (ACTIVE→DELETED). Independent axes.
4. **Actions driven by lifecycle** — getActions(status) returns specific set per state. BOOKED=6, CHECKED_IN/RESCHEDULED=0, NO_SHOW=2, CANCELLED=1.
5. **Rebook ≠ Reschedule** — rebook from NO_SHOW/CANCELLED creates independent appointment (no chain). Reschedule from BOOKED creates chained appointment.
6. **KPI math must add up** — รอมา + เช็คอิน + ไม่มา = total. Previous 5 cards didn't sum correctly.
7. **Sidebar 30%: calendar + vet workload + timeline** — designed, not implemented. Sidebar reacts to vet filter.

## Honest Feedback

**Friction 1: Scope escalation is predictable but still surprising.** "Compare this table with dashboard" became a 4.5-hour session touching appointment lifecycle, KPI design, flow docs, and sidebar design. The cascading pattern is documented (May 4, May 23) but I still underestimate it every time. Fix: when Un asks "compare X with Y", I should proactively flag "this will likely cascade — shall I scope to table-only, or audit the full page?"

**Friction 2: The COMPLETED state should never have existed.** The flow doc always said lifecycle ends at CHECKED_IN. The prototype had COMPLETED because it was built before the flow doc was written. This is the documentation-as-architecture-tool pattern (May 22 learning): writing the lifecycle doc would have caught this immediately. Fix: always check flow docs before adding states to prototypes. If no flow doc exists, write one first.

**Friction 3: Uncommitted work accumulating.** The vet repo has 8 modified files + 20+ new files (including number format standards) sitting uncommitted. The session ended without committing. The post-commit checklist (CHANGELOG + DS export) also wasn't done. This compounds — each session's uncommitted work makes the next commit harder to describe. Fix: commit before the grill starts, commit after the grill ends, never leave both in flight.

## Lessons Learned

1. **Alignment work is a probe, not a patch** — comparing two tables reveals the full accumulated drift of a page. Budget for cascading scope when starting alignment work.
2. **Flow doc is the state machine authority** — prototypes that add states without consulting the flow doc will drift. Write the flow doc first, then prototype.
3. **KPI cards must be mathematically auditable** — if breakdown rows don't sum to the headline number, the design is wrong. Un caught "15 - 13 = 2, where are they?" immediately.
4. **"ไม่มา" is better than "ไม่สำเร็จ" for grouping terminal states** — because RESCHEDULED isn't "unsuccessful", it's a proactive change. Label choice matters for trust.
5. **Vet workload fills sidebar dead space with genuinely unique value** — it's the only information not available in the table or KPI cards, and it answers a real workflow question ("which vet is free?").
