## Honest Feedback

**Friction 1: Grill format mismatch**. The user initially gave me the HN format as `HN-69-05-001` (with dash after prefix) but the handoff doc says `HN69-05-001` (no dash). I should have read the handoff doc FIRST and caught the discrepancy immediately, instead of grilling based on wrong assumptions and requiring a full re-grill. Cost: ~15 minutes of wasted grill questions.

**Friction 2: Agent output verification**. The Chrome agent created 10 HTML files but put broken cross-references to the gitignored `docs/` directory. I verified the files existed and checked for `file://` paths, but didn't think to check whether the *target* of relative links was actually in git. The user's "re-check paths" request caught this. I need a mental checklist: (1) no absolute paths, (2) no file:// paths, (3) link targets exist in git, (4) link targets are in public/ or served by the framework.

**Friction 3: Symlink debugging rabbit hole**. When verifying the symlink, rtk's `ls` output was filtering/truncating results which made it look like the symlink was empty when it was actually working. I spent several rounds of `ls`, `file`, `stat`, `find -L` debugging what turned out to be a display artifact. Should have gone straight to `stat -L` and `find -L ... | wc -l` to confirm resolution.

## Lessons Learned

1. **Read the source of truth BEFORE grilling** — handoff-overview.html had 15 locked decisions (D1-D15). Grilling without reading it first led to redundant questions and wrong assumptions.
2. **Symlink in public/ must be relative** — absolute symlinks work locally but break for every other developer. Always use relative paths for git-tracked symlinks.
3. **Gitignored dirs can't be link targets** — `docs/` was gitignored, so cross-references from tracked files to it will 404 for others. Convert to text references with "open from repo root" note.
4. **"Grill-then-parallel-execute" ratio 3:1 is healthy** — 45 min grill + research, 15 min execution per entity. Don't rush to execute before decisions are locked.
