## Honest Feedback

**Friction 1: I don't read existing code thoroughly enough before writing.** The queue page fiasco was entirely preventable. I had the pickup-queue-to-opd page right there — same folder, same project. I should have read it end-to-end and copied. Instead I wrote from imagination and got every convention wrong. This is the second time (first was the sidebar incident). The memory was already there but I didn't follow it.

**Friction 2: Inconsistency compounds silently.** Once the dashboard had different terminology (`สัตว์` vs `สัตว์เลี้ยง`, `ผู้ดูแล` vs `สัตวแพทย์`), every new page that cloned from it would inherit the wrong terms. The user caught these one by one — each one a small fix, but collectively they represent a systemic failure to maintain a single source of truth.

**Friction 3: Playwright screenshots don't show hydrated state.** `useMediaQuery` returns `false` on initial render, so every Playwright screenshot showed the narrow-mode UI even at 1920px. This wasted time — I kept thinking the responsive code wasn't working when it was. I should have warned the user earlier that Playwright captures pre-hydration state and asked them to verify in the real browser.

## Lessons Learned

1. **Clone, don't create.** When building a new prototype page that shares the same data/layout as an existing one, literally copy the file and modify. Never write from memory.
2. **Terminology is a contract.** Column headers, button labels, status names, HN format — these are not style preferences, they're contracts across pages. One deviation creates cascading inconsistency.
3. **Brand color hierarchy must be explicit.** Pink=identity, orange=action, yellow=completion — writing this down in memory prevents the "orange row buttons" mistake from recurring.
4. **Responsive buttons: icon+label at wide, icon-only at narrow.** This educates users at full-screen and saves space on narrow — better than choosing one or the other.
5. **`line-clamp` needs inline styles in table cells.** Tailwind `line-clamp-2` class doesn't work inside `<td>` — use `-webkit-box` inline styles with a wrapper div.
