## AI Diary

This session was a lesson in compounding scope. What started as a simple "why is the sidebar still showing" turned into a full design system overhaul. The sidebar fix was clean — three `!isOPD` guards, done in minutes. But then each layer revealed the next: if we remove the sidebar, the top bar needs to change. If the top bar changes, the DS needs updating. If the DS is updating, why not add a changelog? If there's a changelog, make it the landing page. If there's a download, make the markdown comprehensive enough for AI handoff.

The debugging detour with the stale dev server cost me 30+ minutes. The "missing required error components" message was misleading — it wasn't my code breaking, it was the dev server needing a restart after too many hot-reload cycles. I should have restarted earlier instead of chasing phantom errors in the JSX.

The most valuable moment was Un's feedback: "always update DS, don't wait me to ask." This is the PM speaking — they want the design system to be a living document that stays in sync automatically, not a separate maintenance task. That reframes every future UI change: the DS update is part of the definition of done, not an afterthought.

The comprehensive markdown export is the real deliverable here. When a developer downloads that .md and feeds it to their AI, they get every color token, every button variant with its decision tree, every Thai terminology mapping with "do not use" warnings. That's the handoff Un is building toward — the DS as a machine-readable spec, not just a visual reference.

## Honest Feedback

**Friction 1: Dev server state corruption.** The biggest time sink was the dev server going into a bad state and returning 404/500 for pages that had no code errors. I burned 30+ minutes trying `curl`, `npx next build`, checking RTK logs, stashing/unstashing changes — all because the server needed a simple restart. Next time: if a page suddenly returns 500 and `git stash` + test still fails, restart the server immediately.

**Friction 2: RTK filtering build output.** When I ran `npx next build --no-lint`, RTK consumed the actual compilation error message and only showed "Errors: 1". I had to bypass RTK entirely (`/usr/local/bin/npx`) to see the real error was in `queue/page.tsx`, not my code. RTK's token savings are real but swallowing error details is a sharp edge.

**Friction 3: Scope creep without checkpoints.** The session went from "fix sidebar" to "overhaul DS with changelog + markdown export + comprehensive spec" without a pause to scope-check. Each step was reasonable but the total cost was high. A mid-session "is this still the priority?" checkpoint would have helped — maybe the comprehensive markdown could have been a follow-up session.

## Lessons Learned

1. **Dev server restart is the first diagnostic** — if pages return 500 after code changes that look correct, restart before debugging
2. **DS update is definition of done** — every UI pattern change includes its DS reflection, proactively
3. **Markdown export serves AI handoff** — the DS .md is a machine-readable spec for developer AI tools, not just human reference
