## Honest Feedback

**1. Session fragmentation**: This work happened across 4 separate `/exit` + resume cycles. Each time I lost conversational context and had to re-orient. The actual work was maybe 20 minutes of action spread across an hour of session management. For quick config changes, staying in one session would be faster — but I understand the user was testing whether changes took effect between restarts.

**2. The verbose setting discovery was slow**: I didn't immediately know that Claude Code's built-in token counter was controlled by `verbose: true`. I had to spawn a research subagent to find it. This should be more discoverable — or I should have checked settings.json more carefully first, since `verbose: true` was right there.

**3. maw command verification was manual**: I tested commands one by one (`workon`, `wake`, `team`, etc.) with `--help` flags, which sometimes confused maw's argument parser (it tried to resolve `--help` as an oracle name). A `maw help commands` or `maw --list-commands` would have been cleaner. The `maw --help` output ultimately gave the full picture.

## Lessons Learned

- maw v26.5.2 moved core Oracle commands (ls, wake, kill, peek, etc.) into bundled plugins — no registry dependency
- Claude Code's `verbose: true` adds a built-in token counter to the status area — redundant with custom statusline context bars
- When upgrading tools installed from GitHub, remember to use `bun add -g pkg@github:org/repo#tag` format, not npm-style
