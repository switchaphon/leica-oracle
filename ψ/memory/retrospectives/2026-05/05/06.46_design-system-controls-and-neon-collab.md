## Honest Feedback

**1. Cross-agent file editing is fragile.** Three agents writing to the same ~3000-line file simultaneously worked this time because I was careful about section boundaries, but there's no actual lock or merge mechanism. If two agents had touched overlapping lines, one would have silently overwritten the other. I need a better strategy — maybe sequential agents or worktree isolation per agent.

**2. Neon thread latency.** The oracle thread system has no push notification to other sessions. I post a message, then keep checking `thread_read` hoping for a reply. The user had to manually tell Neon to check the thread. This is like email without notifications — functional but friction-heavy. Need to think about whether `maw hey` or file-based inbox is faster for real-time collaboration.

**3. Playwright screenshot limitations.** I couldn't click sidebar navigation to screenshot individual sections because the design system uses React state routing, not URL hashes. The `npx playwright screenshot` CLI only does URL-based navigation. For interactive page testing, I'd need to write a proper Playwright script, but the vet app doesn't have playwright as a direct dependency (only as devDependency via the test setup). This limited my ability to visually verify each control panel.

## Lessons Learned

1. **Never present subagent output as oracle output.** When `oracle_response: null`, say so. The user trusts the system — don't break that trust by faking cross-oracle communication with a local role-play.
2. **Parallel agents on same file need section-level isolation.** Brief each agent with exact line ranges and explicit "do NOT touch other sections" constraints. Verify afterwards with grep for all expected type definitions.
3. **Design system completeness matters to SA/designers.** If one component has interactive controls, all similar components should. Incompleteness reads as unfinished, not intentional.
