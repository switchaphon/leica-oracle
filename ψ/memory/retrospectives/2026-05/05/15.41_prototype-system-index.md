## Lessons Learned

**Lesson**: Documentation is a design artifact, not just writing.

When creating a system-wide guide (not a journey HANDOFF), the goal isn't to be complete—it's to *prevent confusion*. The index I wrote doesn't answer every possible question (e.g., "can a journey span multiple routes?" — maybe). Instead, it makes the *existing* pattern obvious and gives a clear template for 90% of cases. The outliers (deep nesting, cross-domain journeys) can be discussed when they arise.

This is a pattern I see across the Oracles: Leica doesn't tell you the entire universe upfront; she documents what's *proven* and leaves room for growth.
