## AI Diary

This session felt like conducting an orchestra. Eight phases of the PRP, each with clear dependencies, parallelized into four waves — and the execution went remarkably clean. Zero TypeScript errors at every checkpoint. The agent delegation worked well: Phase 1 (doc) and Phase 7 (routes) ran in background while I handled the critical Phase 4 extraction directly. The Chrome agents for Phases 5 and 6 produced working selectors on first pass.

The three bugs that surfaced were all integration-level: the prototype index not registering new flows (obvious miss in the plan), the quick order dropdowns missing the "สั่งประจำ"/"สั่งล่าสุด" sections (the subagents didn't know about this pattern since I only showed them LabTestSelector structure, not the full dropdown content), and the flex layout clipping bug where the `data-guide` wrapper broke the flex chain. That last one was satisfying to diagnose — the symptom was "4 items counted but only 3 visible" and the root cause was a plain div interrupting a flex-1/min-h-0 chain.

Then came the pivot. Un looked at the current implementation and said: actually, I want ezyVet-style body maps where the vet annotates directly on the anatomy. The grill session was productive — 9 decisions in about 30 minutes, all in Thai which felt more natural for clinical UX decisions. The key insight was "region-first, not view-first" — vets think "I need Thorax X-ray" before they think about which projection angle. The mockup screenshot (Thorax region card with RT Lat and DV body map cards in a grid) crystallized the vision perfectly.

The body map PRP is written and ready. Next session is a significant rewrite of XRayStudySelector with traced SVGs from ezyVet PNGs. I'm concerned about SVG tracing quality — the ezyVet maps are detailed line art and simplifying them into `<path>` elements while keeping them recognizable will be the hardest part.

## Honest Feedback

**Friction 1: Dev server unreachable from sandbox.** I couldn't verify any of the work in a browser. Every curl to localhost:3002 timed out. This means all visual verification is deferred to Un. For a UI prototype session, this is a significant gap — I'm shipping code I can't visually confirm. The TypeScript compile checks catch structural errors but not layout bugs (like the flex clipping issue that only Un caught via screenshot).

**Friction 2: Agent prompts need more context about sibling patterns.** Both the X-ray and Ultrasound subagents produced working selectors but missed the "สั่งประจำ"/"สั่งล่าสุด" dropdown sections because I only briefed them on the component structure, not on the full UX pattern expectations. I should have included "match the lab selector's shortcut dropdown exactly: สั่งประจำ → สั่งล่าสุดของ {petname} → ชุดตรวจ" in the prompt.

**Friction 3: The PRP-then-grill-then-revise cycle.** The original PRP had the geometric rectangle anatomy diagram. After building it, Un wanted ezyVet body maps — a fundamentally different interaction model. The grill should have happened BEFORE execution, not after. In this case the PRP was pre-grilled (from a previous session) but the body map idea emerged from seeing the actual implementation. Lesson: show mockups early, build late.

## Lessons Learned

1. **flex-1 min-h-0 chains break when a plain div wrapper is inserted** — any `data-guide` or similar attribute wrapper in a flex layout must also carry `flex-1 min-h-0 flex flex-col` or it clips overflow content invisibly
2. **Agent briefs need UX pattern examples, not just component structure** — when delegating UI components, include screenshots or explicit "match this dropdown exactly" references
3. **Grill body map/annotation UX before building anatomy components** — the ezyVet reference changed everything; should have been part of the original PRP grill
