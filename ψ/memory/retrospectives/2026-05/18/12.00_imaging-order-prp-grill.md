## AI Diary

This was one of those sessions where I felt like a proper PM running a design review, not just a code generator. The /grill-me skill forced a structured walk through every design branch, and each question surfaced something I wouldn't have caught if I'd just started coding.

The pivotal moment was when Un shared that real yellow hospital X-ray request form. That single photo invalidated half of what the Figma wireframes suggested — the Figma had 6 regions, the real form had 8 (Joint, Bone, Vertebrae were missing), and the projection catalog was more nuanced than I'd assumed. The research agent found AAHA benchmarks, Thai market norms, and even how ezyVet/Cornerstone handle it. That gave Un confidence to choose the tiered model, and it gave me confidence the data model would work.

What made this session productive was the rhythm: explore → question → research → decide → next. No code written, but 15 hard decisions resolved. The PRP is dense and actionable — the next session should be pure execution.

I'm proud that the plan properly handles the two different pricing granularities (X-ray tiered vs ultrasound flat) within the same inventory schema, and that the OrderSummaryPane extraction means we're not duplicating 130 lines of right-pane code across three selectors.

## Honest Feedback

1. **Figma context extraction was painful**: Each wireframe returned 1-2M characters of design context, way too large for direct consumption. Had to delegate to 3 separate agents just to summarize them. The Figma MCP plugin should have a "summary mode" or at least a way to request only text/labels without full code generation. This burned ~15 minutes and a lot of tokens.

2. **Data model docs were too large to read**: Both DATA_DICTIONARY_INVENTORY.md (34K tokens) and LAB_ORDER_DATA_MODEL.md (28K tokens) exceeded the 25K read limit. Had to chunk-read them. For future sessions, these docs should either have a TL;DR section at the top, or I should create condensed reference sheets in memory.

3. **Plan mode friction with /grill-me**: The plan mode system prompt says "read-only, no edits" but /grill-me is inherently a design discussion that produces a document. I ended up writing the PRP to the prototype folder (user's request) while technically in plan mode. The write was allowed because the user explicitly asked, but the flow felt awkward — plan mode wants me to write to `.claude/plans/` but the user wanted `/prototype/prp/`.

## Lessons Learned

1. **Real artifacts beat wireframes**: A single photo of a real hospital form was more informative than 3 Figma wireframes totaling 5M characters. Always ask for real-world references when designing clinical workflows.
2. **Research before deciding pricing**: Don't assume domain pricing models — vet imaging pricing is fundamentally different from lab test pricing (tiered study vs per-item). Web research saved us from building the wrong model.
3. **Grilling resolves ambiguity upfront**: 8 rounds of questions in 40 minutes prevented what would have been days of rework during implementation. The /grill-me skill pays for itself.
