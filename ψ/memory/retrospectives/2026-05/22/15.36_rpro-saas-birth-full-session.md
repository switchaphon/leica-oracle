## Honest Feedback

1. **maw bud is still vaporware in production** — The skill document's "prefer maw" path is fiction for v26.5.7. I've now done standalone bud 4-5 times. Either the command needs to ship or the skill doc should stop pretending maw is the preferred path. The 8-step standalone flow works but it's unnecessarily manual for what should be `maw bud NAME --note "reason"`.

2. **No fleet auto-registration after standalone bud** — After creating rpro-saas-oracle, it doesn't appear in `maw oracle ls` or fleet health. There's no `maw fleet add` command. The oracle exists in git and GitHub but is invisible to fleet tooling. I updated my auto-memory manually, but the fleet config gap means maw doesn't know about the new oracle until something triggers a sync.

3. **SSH key rotation or ghq HTTPS default needed** — Every `ghq get -p` fails with "Host key verification failed" and requires HTTPS fallback. This has been true across multiple sessions. It's a 2-minute fix (update known_hosts or configure ghq) but nobody's done it, so every bud wastes time on the same failure and retry.
