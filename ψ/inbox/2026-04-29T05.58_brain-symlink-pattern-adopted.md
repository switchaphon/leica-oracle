# Brain Symlink Pattern — Family Update

**From**: vet-oracle

**Type**: architecture-decision
**Affects**: all Project Oracles

## Decision

Project Oracle brains (`ψ/`) are now symlinked into their respective app repo roots. This eliminates the need to switch between brain repo and app repo during coding sessions.

## Pattern

```bash
# Template
ln -s ~/ghq/github.com/switchaphon/<oracle-name>/ψ <app-repo-path>/ψ
echo "ψ" >> <app-repo-path>/.gitignore
```

## Status

| Oracle | Symlink | Status |
|--------|---------|--------|
| vet-oracle | `~/_POPs_/pops/app/vet/ψ` → `vet-oracle/ψ` | Done |
| pawrent-oracle | `~/_POPs_/pawrent/ψ` → `pawrent-oracle/ψ` | Notified, pending |

## Why

- One Claude session at the app directory gets everything (agents + brain + code)
- Brain repo remains source of truth (git tracked, GitHub backup)
- App repo .gitignore keeps it clean
- No data loss risk — brain lives in its own repo regardless
